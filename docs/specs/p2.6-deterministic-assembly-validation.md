> 变更: 新增 `validate-assembly.mjs` 确定性脚本，替代 Phase 3 的 `assemble-reviewer` LLM 子代理调用

## 1. 目标与范围

### 1.1 问题

Phase 3 当前派遣 `assemble-reviewer` 子代理审查合并后的 `assembled-graph.json`。该子代理执行的所有检查项均为结构化、规则化的操作：

- 节点 ID 唯一性
- 必填字段完整性（id, type, name, summary, tags）
- 边引用完整性（source/target 存在于节点集）
- imports 边覆盖率（对比 importMap）
- 重复节点/边检测

这些检查不涉及语义理解或判断，LLM 调用耗时 30-60 秒，属于冗余开销。Phase 6 的 `ua-inline-validate.cjs` 已证明同类结构化检查可确定性完成。

### 1.2 目标

- 消除 Phase 3 的 LLM 调用，节省 30-60 秒
- 确定性验证比 LLM 更可靠（零幻觉、零遗漏、确定性输出）
- 新增复杂度规范化自动修复能力（原 assemble-reviewer 的 Step 2 部分）
- 新增孤立节点检测（原 `ua-inline-validate.cjs` 已有，本脚本统一覆盖）

### 1.3 范围

**包含**：
- 新增 `validate-assembly.mjs` 确定性验证脚本
- 修改 SKILL.md Phase 3 流程：替换子代理调度为脚本调用
- 新增测试文件 `test_validate_assembly.test.mjs`

**不包含**：
- assemble-reviewer 的 Step 2（调查 "Could not fix" 问题）— 这些问题已由 `merge-batch-graphs.py` 的规范化逻辑处理
- assemble-reviewer 的 Step 3（跨批次边补充）— 已由优化 F（imports 边确定性生成）覆盖
- assemble-reviewer.md 文件删除 — 保留作为历史参考，仅不再调度
- Phase 6 的 `ua-inline-validate.cjs` 变更 — 该脚本在最终验证阶段仍有价值（检查 layers/tour 结构），本脚本仅覆盖 Phase 3 的组装验证

### 1.4 约束

- 不引入新的外部依赖
- 脚本遵循现有模式：CLI 参数、stderr 日志、JSON 输出
- 验证结果格式与 `ua-inline-validate.cjs` 兼容（issues/warnings/stats 结构）
- 自动修复操作仅限复杂度规范化（低风险、可逆），不自动删除或创建节点/边
- 退出码：0 = 成功（无论是否发现 issues），1 = 脚本执行错误

## 3. 详细设计

### 3.1 新脚本：`validate-assembly.mjs`

**文件路径**: `skills/understand/validate-assembly.mjs`

#### 3.1.1 CLI 接口

```bash
node <SKILL_DIR>/validate-assembly.mjs <assembledGraphPath> <importMapPath> <outputPath>
```

| 参数 | 说明 |
|------|------|
| `assembledGraphPath` | `assembled-graph.json` 文件路径 |
| `importMapPath` | `import-map.json` 文件路径 |
| `outputPath` | 验证结果输出路径 |

#### 3.1.2 处理流程

```
1. 读取 assembled-graph.json
2. 读取 import-map.json
3. 执行验证检查（7 项）
4. 执行自动修复（复杂度规范化）
5. 如有修复，回写 assembled-graph.json
6. 写入验证结果到 outputPath
7. 输出摘要到 stderr
```

#### 3.1.3 验证检查项

##### 检查 a：节点 ID 唯一性

```javascript
/**
 * 遍历所有节点，检测重复 ID。
 * 重复 ID 会导致后续合并和引用解析错误。
 */
const seen = new Map(); // id → firstIndex
graph.nodes.forEach((node, i) => {
  if (!node.id) return; // 缺失 ID 由检查 b 处理
  if (seen.has(node.id)) {
    issues.push(`Duplicate node ID '${node.id}' at indices ${seen.get(node.id)} and ${i}`);
  } else {
    seen.set(node.id, i);
  }
});
```

##### 检查 b：必填字段完整性

```javascript
/**
 * 每个节点必须包含 id, type, name, summary, tags 五个字段。
 * 与 ua-inline-validate.cjs 的检查规则一致。
 */
const REQUIRED_FIELDS = ['id', 'type', 'name', 'summary', 'tags'];

graph.nodes.forEach((node, i) => {
  for (const field of REQUIRED_FIELDS) {
    if (node[field] === undefined || node[field] === null) {
      issues.push(`Node[${i}] '${node.id || '<no-id>'}' missing ${field}`);
    }
  }
});
```

##### 检查 c：边引用完整性

```javascript
/**
 * 每条边的 source 和 target 必须存在于节点集中。
 * 悬空边会导致知识图谱渲染和导航错误。
 */
const nodeIds = new Set(graph.nodes.map(n => n.id).filter(Boolean));

graph.edges.forEach((edge, i) => {
  if (!nodeIds.has(edge.source)) {
    issues.push(`Edge[${i}] source '${edge.source}' not found in node set`);
  }
  if (!nodeIds.has(edge.target)) {
    issues.push(`Edge[${i}] target '${edge.target}' not found in node set`);
  }
});
```

##### 检查 d：imports 边覆盖率

```javascript
/**
 * 对比 importMap 中的导入关系与图中实际存在的 imports 边，
 * 计算覆盖率。覆盖率低于 100% 表示有导入关系未被表示为边。
 *
 * 注意：此检查在优化 F（imports 边确定性生成）实施后，
 * 覆盖率应为 100%。此检查作为安全网保留。
 */
function checkImportsCoverage(graph, importMap) {
  // 1. 构建 filePath → nodeId 映射（仅映射 type="file" 的节点）
  const filePathToNodeId = new Map();
  for (const node of graph.nodes) {
    if (node.type === 'file' && node.filePath) {
      filePathToNodeId.set(node.filePath, node.id);
    }
  }

  // 2. 统计 importMap 中可映射的导入对数
  let totalMappable = 0;
  const mappablePairs = new Set(); // "sourceId→targetId"
  for (const [sourcePath, targetPaths] of Object.entries(importMap)) {
    const sourceId = filePathToNodeId.get(sourcePath);
    if (!sourceId) continue;
    for (const targetPath of targetPaths) {
      const targetId = filePathToNodeId.get(targetPath);
      if (!targetId) continue;
      totalMappable++;
      mappablePairs.add(`${sourceId}→${targetId}`);
    }
  }

  // 3. 统计图中已有的 imports 边
  const existingImports = new Set();
  for (const edge of graph.edges) {
    if (edge.type === 'imports') {
      existingImports.add(`${edge.source}→${edge.target}`);
    }
  }

  // 4. 计算覆盖率
  let covered = 0;
  for (const pair of mappablePairs) {
    if (existingImports.has(pair)) covered++;
  }

  const importsCoverage = totalMappable > 0 ? covered / totalMappable : 1;

  // 5. 报告
  if (importsCoverage < 1) {
    warnings.push(
      `Imports edge coverage: ${(importsCoverage * 100).toFixed(1)}% ` +
      `(${covered}/${totalMappable} mappable import pairs have edges)`
    );
  }

  return importsCoverage;
}
```

##### 检查 e：复杂度规范化

```javascript
/**
 * 将非标准复杂度值映射到标准值。
 * 标准值：simple, moderate, complex
 *
 * 映射规则（与 merge-batch-graphs.py 的 NORMALIZE_COMPLEXITY 一致）：
 *   low / easy → simple
 *   medium / intermediate → moderate
 *   high / hard / difficult → complex
 *
 * 此操作为自动修复：修改 assembled-graph.json 中的节点复杂度值。
 */
const COMPLEXITY_MAP = {
  // → simple
  'low': 'simple',
  'easy': 'simple',
  // → moderate
  'medium': 'moderate',
  'intermediate': 'moderate',
  // → complex
  'high': 'complex',
  'hard': 'complex',
  'difficult': 'complex',
};

const VALID_COMPLEXITY = new Set(['simple', 'moderate', 'complex']);

let fixedComplexity = 0;
graph.nodes.forEach((node) => {
  if (!node.complexity) return;
  const lower = String(node.complexity).toLowerCase();
  if (VALID_COMPLEXITY.has(lower)) {
    node.complexity = lower; // 确保小写
    return;
  }
  if (COMPLEXITY_MAP[lower]) {
    node.complexity = COMPLEXITY_MAP[lower];
    fixedComplexity++;
  }
  // 未知复杂度值不修改，由 issues 报告
});
```

##### 检查 f：重复节点/边检测

```javascript
/**
 * 检测语义重复的节点和边。
 * 节点重复：相同 (type, name, filePath) 三元组
 * 边重复：相同 (source, target, type) 三元组
 *
 * 注意：ID 重复由检查 a 处理，此处检测的是 ID 不同但语义相同的重复。
 */
function checkDuplicates(graph, issues, warnings) {
  // 节点语义重复
  const nodeSignature = (n) => `${n.type}::${n.name}::${n.filePath || ''}`;
  const nodeSigs = new Map(); // signature → [indices]
  graph.nodes.forEach((node, i) => {
    const sig = nodeSignature(node);
    if (!nodeSigs.has(sig)) nodeSigs.set(sig, []);
    nodeSigs.get(sig).push(i);
  });
  let duplicateNodes = 0;
  for (const [sig, indices] of nodeSigs) {
    if (indices.length > 1) {
      duplicateNodes += indices.length - 1;
      warnings.push(
        `Duplicate node signature '${sig}' at indices [${indices.join(', ')}]`
      );
    }
  }

  // 边重复
  const edgeSignature = (e) => `${e.source}→${e.target}::${e.type}`;
  const edgeSigs = new Map();
  graph.edges.forEach((edge, i) => {
    const sig = edgeSignature(edge);
    if (!edgeSigs.has(sig)) edgeSigs.set(sig, []);
    edgeSigs.get(sig).push(i);
  });
  let duplicateEdges = 0;
  for (const [sig, indices] of edgeSigs) {
    if (indices.length > 1) {
      duplicateEdges += indices.length - 1;
      warnings.push(
        `Duplicate edge '${sig}' at indices [${indices.join(', ')}]`
      );
    }
  }

  return { duplicateNodes, duplicateEdges };
}
```

##### 检查 g：孤立节点检测

```javascript
/**
 * 检测无任何边连接的节点。
 * 孤立节点可能是分析遗漏或文件未被任何其他文件引用。
 * 此为警告级别，不阻塞流程。
 */
const withEdges = new Set([
  ...graph.edges.map(e => e.source),
  ...graph.edges.map(e => e.target),
]);

let orphanNodes = 0;
graph.nodes.forEach((node) => {
  if (node.id && !withEdges.has(node.id)) {
    orphanNodes++;
    warnings.push(`Node '${node.id}' has no edges (orphan)`);
  }
});
```

#### 3.1.4 自动修复逻辑

仅复杂度规范化为自动修复操作：

```javascript
/**
 * 自动修复流程：
 * 1. 执行复杂度规范化（检查 e）
 * 2. 如有修复（fixedComplexity > 0），回写 assembled-graph.json
 * 3. 在 stats 中记录修复数量
 *
 * 回写时保持 JSON 格式化（2 空格缩进），与 merge-batch-graphs.py 输出一致。
 */
if (fixedComplexity > 0) {
  writeFileSync(assembledGraphPath, JSON.stringify(graph, null, 2));
  process.stderr.write(`Fixed ${fixedComplexity} complexity value(s)\n`);
}
```

#### 3.1.8 完整脚本骨架

```javascript
#!/usr/bin/env node

/**
 * validate-assembly.mjs — 确定性组装验证脚本
 *
 * 替代 Phase 3 的 assemble-reviewer LLM 子代理调用。
 * 对 assembled-graph.json 执行结构化验证检查，
 * 自动修复复杂度值，输出验证结果。
 *
 * 用法: node validate-assembly.mjs <assembledGraphPath> <importMapPath> <outputPath>
 */

import { readFileSync, writeFileSync } from 'node:fs';
import { resolve } from 'node:path';

const [,, assembledGraphPath, importMapPath, outputPath] = process.argv;

if (!assembledGraphPath || !importMapPath || !outputPath) {
  process.stderr.write('Usage: validate-assembly.mjs <assembledGraphPath> <importMapPath> <outputPath>\n');
  process.exit(1);
}

try {
  // 1. 读取输入
  const graph = JSON.parse(readFileSync(resolve(assembledGraphPath), 'utf-8'));
  const importMapData = JSON.parse(readFileSync(resolve(importMapPath), 'utf-8'));
  const importMap = importMapData.importMap || importMapData;

  // 2. 初始化
  const issues = [];
  const warnings = [];

  // 3. 执行检查 a-g
  // ... (各检查项实现见上文)

  // 4. 自动修复
  if (fixedComplexity > 0) {
    writeFileSync(resolve(assembledGraphPath), JSON.stringify(graph, null, 2));
    process.stderr.write(`Fixed ${fixedComplexity} complexity value(s)\n`);
  }

  // 5. 写入输出
  const result = {
    valid: issues.length === 0,
    issues,
    warnings,
    stats: {
      totalNodes: graph.nodes.length,
      totalEdges: graph.edges.length,
      importsCoverage,
      orphanNodes,
      duplicateNodes,
      duplicateEdges,
      fixedComplexity,
    },
  };

  writeFileSync(resolve(outputPath), JSON.stringify(result, null, 2));

  // 6. stderr 摘要
  process.stderr.write([
    'Validating assembled graph...',
    `  Nodes: ${result.stats.totalNodes}, Edges: ${result.stats.totalEdges}`,
    `  Imports coverage: ${(importsCoverage * 100).toFixed(1)}%`,
    `  Orphan nodes: ${orphanNodes}`,
    `  Duplicate nodes: ${duplicateNodes}, Duplicate edges: ${duplicateEdges}`,
    fixedComplexity > 0 ? `  Fixed ${fixedComplexity} complexity value(s)` : '',
    `Validation complete: ${issues.length} issue(s), ${warnings.length} warning(s)`,
  ].filter(Boolean).join('\n') + '\n');

} catch (err) {
  process.stderr.write(`Error: ${err.message}\n`);
  process.exit(1);
}
```

```javascript
}

/**
 * 构造标准节点。
 */
function makeNode(overrides = {}) {
  return {
    id: 'file:src/index.ts',
    type: 'file',
    name: 'index.ts',
    summary: 'Main entry point',
    tags: ['entry'],
    filePath: 'src/index.ts',
    complexity: 'moderate',
    ...overrides,
  };
}

/**
 * 构造标准边。
 */
function makeEdge(overrides = {}) {
  return {
    source: 'file:src/index.ts',
    target: 'file:src/utils.ts',
    type: 'imports',
    direction: 'forward',
    weight: 0.7,
    ...overrides,
  };
}

/**
 * 构造 import-map.json 测试数据。
 */
function makeImportMap(map = {}) {
  return { importMap: map };
}

/**
 * 运行验证脚本，返回结果。
 */
function runValidate(graphPath, importMapPath, outputPath) {
  const result = spawnSync('node', [SCRIPT, graphPath, importMapPath, outputPath], {
    encoding: 'utf-8',
  });
  let output = null;
  try {
    output = JSON.parse(readFileSync(outputPath, 'utf-8'));
  } catch { /* output missing on failure */ }
  return { status: result.status, stderr: result.stderr, output };
}
```
