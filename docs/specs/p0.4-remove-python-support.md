# P0.4: 移除 Python 语言支持 — 详细设计规格

> 版本: 1.1
> 日期: 2026-06-04
> 状态: 已评审通过
> 变更: v1.1 — 修复评审问题：补充遗漏文件（vite.config.ts、根 package.json、assemble-reviewer.md、extract-domain-context.py 详细清理点），所有路径添加 `understand-anything-trae-plugin/` 前缀

## 1. 目标与范围

### 1.1 目标

从 `understand-anything-trae-plugin` 中**彻底移除** Python 语言的所有分析能力，包括：
- 语言配置注册
- 结构化提取器
- Import 解析逻辑
- 测试文件配对逻辑
- LLM 提示词注入
- 框架配置（Django / FastAPI / Flask）
- 所有相关测试用例和文档引用

### 1.2 不在范围内

以下文件**不删除**，但需清理其中的 Python 特定内容：

| 文件 | 说明 |
|------|------|
| `skills/understand/merge-subdomain-graphs.py` | 独立 Python 运行时脚本，非"对 Python 语言的分析" |
| `skills/understand-domain/extract-domain-context.py` | 同上，独立 Python 脚本，仅清理其中 `.py` 扩展名映射等 Python 分析相关代码 |

### 1.3 移除后保留的语言支持链

```
TypeScript/JavaScript → LanguageConfig + TreeSitter WASM + TypeScriptExtractor + resolveTsJsImport ✅
Vue SFC → LanguageConfig + VueSfcPlugin (@vue/compiler-sfc) + TS tree-sitter ✅
Svelte → LanguageConfig + SveltePlugin (svelte/compiler) + TS tree-sitter ✅
Python → ❌ 完全移除
其他后端语言 → 已在 P0.3 阶段清理（Go/Rust/Java/Kotlin/C#/PHP/Ruby/C/C++）
```

## 4. 风险评估

| 风险项 | 级别 | 缓解措施 |
|--------|------|----------|
| 删除 pythonConfig 导致 `builtinLanguageConfigs` 数组空槽位 | 低 | 数组是动态构建的，删除元素不影响索引 |
| 测试文件中使用 Python 作为示例数据导致测试语义变化 | 低 | 替换为同等复杂度的 TS/JS/Vue/Svelte 示例 |
| `merge-batch-graphs.py` 的 `_TEST_NAME_PATTERNS` 变为空字典 | 低 | 空 dict 时 `is_test_path()` 正确返回 False，符合预期行为 |
| `extract-import-map.mjs` 中 `resolveImport()` 对 `.py` 文件返回空数组 | 低 | 符合"不支持的语言返回空 importMap"的设计契约 |
| 文档中遗漏 Python 引用导致 LLM agent 输出过时信息 | 中 | 阶段 5.5 全局搜索兜底 |
| `extract-domain-context.py` 中 `.py` 扩展名被误删 | 低 | 该文件不在本次删除范围内，仅清理 Python 分析逻辑 |
