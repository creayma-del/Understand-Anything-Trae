# P1.4: React JSX 语义提取器详细设计规格

> 版本: 1.0
> 日期: 2026-06-05
> 状态: 已评审（修复 HIGH 问题后）
> 前置: P0（Vue SFC / Svelte / SCSS 增强）已完成
> 变更: 基于 TypeScriptExtractor 扩展 React 语义识别，增加 tags 字段标记 hooks/组件/Context/HOC 等模式

## 1. 目标与范围

在 TypeScriptExtractor 基础上增加 React 特有模式识别，使项目对 React 代码库的分析能力从"通用 TS/JS 结构提取"升级为"React 语义感知提取"，自动识别组件关系、hooks 调用链、Context 依赖、HOC 包装等模式。

**范围**：
- 新增 `ReactExtractor`，继承 `TypeScriptExtractor`，增量分析 React 模式
- 扩展 `StructuralAnalysis.functions` 类型，增加 `tags` 字段
- 扩展 `CallGraphEntry` 类型，增加 `relationType` 字段
- 替换 `builtinExtractors` 注册，用 `ReactExtractor` 替代 `TypeScriptExtractor`
- 新增 LLM 分析提示词

**不在范围内**：
- CSS-in-JS 深度分析（P1.8 阶段）
- React Native 特有模式
- Next.js 框架配置（独立子项）
- 运行时行为分析

## 2. React 代码模式分析

React 代码具有 5 种核心语义模式，当前 `TypeScriptExtractor` 无法识别：

### 2.1 Hooks 模式

```typescript
// 内置 hooks — 从 'react' 导入
import { useState, useEffect, useCallback } from 'react';
const [count, setCount] = useState(0);
useEffect(() => { /* ... */ }, []);

// 自定义 hooks — use 前缀 + PascalCase 后续
function useCustomHook() {
  const [value] = useState(null);
  return value;
}
```

**识别特征**：
- 内置 hooks：函数名匹配 `REACT_HOOKS` 列表 + 从 `'react'` 导入
- 自定义 hooks：函数名匹配 `use[A-Z]` 模式 + 从 hook 文件导入或当前文件内定义

### 2.2 组件声明模式

```typescript
// 函数声明组件
function MyComponent() { return <div />; }

// 箭头函数组件
const MyComponent = () => <div />;

// React.FC 类型标注
const MyComponent: React.FC<Props> = () => <div />;

// forwardRef 组件
const MyComponent = forwardRef((props, ref) => <div ref={ref} />);

// 类组件
class MyComponent extends React.Component { render() { return <div />; } }
```

**识别特征**：
- 函数名 PascalCase（`^[A-Z][a-zA-Z0-9]*$`）
- 返回值包含 JSX（`jsx_element` / `jsx_self_closing_element` 节点）
- 类型标注包含 `React.FC` / `React.FunctionComponent` / `React.SFC`
- 被 `forwardRef` / `memo` 包裹

### 2.3 JSX 组件组合模式

```typescript
// 父组件包含子组件
function Parent() {
  return (
    <div>
      <ChildComponent name="foo" />
      <ThemeProvider><App /></ThemeProvider>
    </div>
  );
}
```

**识别特征**：
- JSX 中 `<PascalCase />` 标签（非 HTML 标准标签）
- 子组件名在当前文件 imports 中存在（交叉验证，避免误判 HTML 标签）

### 2.4 Context 关系模式

```typescript
// Context 定义
const ThemeContext = createContext('light');

// Context 消费（hooks 方式）
const theme = useContext(ThemeContext);

// Context 消费（Consumer 方式）
<ThemeContext.Consumer>{value => /* ... */}</ThemeContext.Consumer>

// Context Provider
<ThemeContext.Provider value={theme}><App /></ThemeContext.Provider>
```

**识别特征**：
- `createContext` 调用 → Context 定义
- `useContext` 调用 → Context 消费
- `.Provider` / `.Consumer` JSX 标签 → Context 关系

### 2.5 HOC 包装模式

```typescript
// 经典 HOC
export default withRouter(MyComponent);
export default connect(mapStateToProps, mapDispatchToProps)(MyComponent);

// React.memo
const MemoComponent = memo(MyComponent);

// React.forwardRef
const RefComponent = forwardRef((props, ref) => {});
```

**识别特征**：
- export default 中函数调用包裹标识符
- `memo()` / `forwardRef()` 包裹
- 任意函数调用包裹 PascalCase 标识符（HOC 模式）

## 3. ExtractionResult 类型扩展

### 3.1 现有类型

当前 `StructuralAnalysis.functions` 类型定义（来自 `packages/core/src/types.ts`）：

```typescript
functions: Array<{
  name: string;
  lineRange: [number, number];
  params: string[];
  returnType?: string;
}>;
```

**问题**：缺少语义标签能力，无法区分普通函数、hooks、组件等 React 模式。

### 3.2 扩展方案：增加 tags 字段

**文件**: `packages/core/src/types.ts`

```typescript
functions: Array<{
  name: string;
  lineRange: [number, number];
  params: string[];
  returnType?: string;
  // ★ P1.4 新增: 语义标签
  tags?: string[];
}>;
```

**tags 可选值**：

| 标签 | 含义 | 识别来源 |
|---|---|---|
| `hook` | React hook 函数 | `identifyHooks()` |
| `builtin-hook` | React 内置 hook（useState 等） | `identifyHooks()` |
| `custom-hook` | 自定义 hook（use[A-Z] 模式） | `identifyHooks()` |
| `component` | React 组件 | `identifyComponents()` |
| `function-component` | 函数组件 | `identifyComponents()` |
| `class-component` | 类组件 | `identifyComponents()` |
| `forward-ref` | forwardRef 包裹的组件 | `identifyComponents()` |
| `hoc-wrapped` | HOC 包装的组件 | `identifyHocPatterns()` |
| `context-definition` | Context 定义（createContext） | `identifyContextRelations()` |
| `context-consumer` | Context 消费（useContext） | `identifyContextRelations()` |

**设计决策**：
- `tags` 为可选字段（`string[]`），向后兼容 — 不影响现有 TypeScriptExtractor 和其他提取器
- 标签采用层级结构（如 `hook` 是 `builtin-hook` / `custom-hook` 的父标签），同时添加父标签和子标签
- 多标签共存：一个函数可以同时是 `component` + `hoc-wrapped`

### 3.3 CallGraphEntry 扩展

**文件**: `packages/core/src/types.ts`

```typescript
export interface CallGraphEntry {
  caller: string;
  callee: string;
  lineNumber: number;
  // ★ P1.4 新增: 关系类型
  relationType?: 'call' | 'contains' | 'depends_on';
}
```

**relationType 可选值**：

| 值 | 含义 | 生成来源 |
|---|---|---|
| `call` | 普通函数调用（默认值） | TypeScriptExtractor 原有逻辑 |
| `contains` | 父组件包含子组件 | `identifyJsxComposition()` |
| `depends_on` | 依赖关系（Context 消费→定义、HOC 包装） | `identifyContextRelations()` / `identifyHocPatterns()` |

**设计决策**：
- `relationType` 为可选字段，默认值为 `'call'`（向后兼容）
- 仅 ReactExtractor 增量分析时添加 `contains` / `depends_on`，原有调用图条目不受影响

### 3.4 imports 扩展

**文件**: `packages/core/src/types.ts`

```typescript
imports: Array<{
  source: string;
  specifiers: string[];
  lineNumber: number;
  // ★ P1.4 新增: 导入来源标记
  importKind?: 'module' | 'component' | 'hook' | 'context' | 'hoc';
}>;
```

**importKind 可选值**：

| 值 | 含义 | 识别规则 |
|---|---|---|
| `module` | 普通模块导入（默认值） | 无特殊模式 |
| `component` | 组件导入 | specifier 为 PascalCase + 导入路径为组件文件 |
| `hook` | hook 导入 | specifier 匹配 `use[A-Z]` 或 `REACT_HOOKS` |
| `context` | Context 导入 | specifier 包含 `Context` 后缀或从 Context 文件导入 |
| `hoc` | HOC 导入 | specifier 匹配 HOC 模式（connect / withRouter 等） |

## 4. ReactExtractor 实现

### 4.1 文件: `packages/core/src/plugins/extractors/react-extractor.ts`

```typescript
import type { StructuralAnalysis, CallGraphEntry } from "../../types.js";
import type { LanguageExtractor, TreeSitterNode } from "./types.js";
import { TypeScriptExtractor } from "./typescript-extractor.js";
import { traverse, findChild, findChildren, getStringValue } from "./base-extractor.js";

/**
 * React 内置 hooks 列表
 */
const REACT_HOOKS = new Set([
  'useState', 'useEffect', 'useCallback', 'useMemo', 'useRef',
  'useContext', 'useReducer', 'useLayoutEffect', 'useDebugValue',
  'useDeferredValue', 'useTransition', 'useId', 'useSyncExternalStore',
  'useInsertionEffect', 'useImperativeHandle',
]);

/**
 * React HOC 函数名列表
 */
const REACT_HOCS = new Set([
  'memo', 'forwardRef',
]);

/**
 * React JSX 语义提取器。
 *
 * 继承 TypeScriptExtractor，在基础 TS/JS 结构提取之上
 * 增量分析 React 特有模式：hooks、组件、JSX 组合、Context、HOC。
 */
export class ReactExtractor extends TypeScriptExtractor implements LanguageExtractor {
  readonly languageIds = ['typescript', 'javascript'];

  extractStructure(rootNode: TreeSitterNode): StructuralAnalysis {
    // 阶段 1: 调用父类提取基础结构
    const result = super.extractStructure(rootNode);

    // 阶段 2: 增量分析 React 模式
    this.identifyHooks(result, rootNode);
    this.identifyComponents(result, rootNode);
    this.identifyJsxComposition(result, rootNode);
    this.identifyContextRelations(result, rootNode);
    this.identifyHocPatterns(result, rootNode);

    return result;
  }

  extractCallGraph(rootNode: TreeSitterNode): CallGraphEntry[] {
    // 阶段 1: 调用父类提取基础调用图
    const entries = super.extractCallGraph(rootNode);

    // 阶段 2: 增量分析 React 关系边
    this.addContainsEdges(entries, rootNode);
    this.addDependsOnEdges(entries, rootNode);

    return entries;
  }

  // ---- React 模式识别方法 ----

  /**
   * 识别 hooks 调用模式。
   *
   * 规则:
   * 1. 内置 hook: 函数名在 REACT_HOOKS 集合中 + 从 'react' 导入
   * 2. 自定义 hook: 函数名匹配 use[A-Z] 模式 + 从 hook 文件导入或当前文件定义
   *
   * 交叉验证: 仅当函数名匹配 hook 模式 且 (从 react 导入 或 use[A-Z] 命名)
   *           才标记为 hook，避免误判
   */
  private identifyHooks(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    // 1. 构建从 'react' 导入的 specifiers 集合
    const reactImports = this.collectReactImports(result);

    // 2. 标记函数中的 hooks
    for (const fn of result.functions) {
      if (REACT_HOOKS.has(fn.name) && reactImports.has(fn.name)) {
        // 内置 hook
        fn.tags = [...(fn.tags ?? []), 'hook', 'builtin-hook'];
      } else if (/^use[A-Z]/.test(fn.name)) {
        // 自定义 hook（use[A-Z] 命名约定）
        fn.tags = [...(fn.tags ?? []), 'hook', 'custom-hook'];
      }
    }

    // 3. 标记 imports 中的 hook 导入
    for (const imp of result.imports) {
      if (imp.source === 'react') {
        const hookSpecifiers = imp.specifiers.filter(s => REACT_HOOKS.has(s));
        if (hookSpecifiers.length > 0) {
          imp.importKind = 'hook';
        }
      } else {
        const hookSpecifiers = imp.specifiers.filter(s => /^use[A-Z]/.test(s));
        if (hookSpecifiers.length > 0 && hookSpecifiers.length === imp.specifiers.length) {
          imp.importKind = 'hook';
        }
      }
    }
  }

  /**
   * 识别组件声明。
   *
   * 规则:
   * 1. 函数名 PascalCase + 返回 JSX → function-component
   * 2. 函数名 PascalCase + React.FC / React.FunctionComponent 类型标注 → function-component
   * 3. forwardRef 包裹 → function-component + forward-ref
   * 4. 类名 PascalCase + extends React.Component / React.PureComponent → class-component
   */
  private identifyComponents(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    for (const fn of result.functions) {
      if (!/^[A-Z][a-zA-Z0-9]*$/.test(fn.name)) continue;

      // 检查是否已被标记为 hook（组件和 hook 互斥）
      if (fn.tags?.includes('hook')) continue;

      // 规则 1: PascalCase + 返回 JSX（通过遍历函数体检测 jsx_element / jsx_self_closing_element）
      if (this.functionReturnsJsx(rootNode, fn)) {
        fn.tags = [...(fn.tags ?? []), 'component', 'function-component'];
        continue;
      }

      // 规则 2: React.FC 类型标注
      if (this.hasReactFcAnnotation(rootNode, fn)) {
        fn.tags = [...(fn.tags ?? []), 'component', 'function-component'];
        continue;
      }
    }

    // 规则 3: forwardRef 包裹
    this.identifyForwardRefComponents(result, rootNode);

    // 规则 4: 类组件
    for (const cls of result.classes) {
      if (!/^[A-Z][a-zA-Z0-9]*$/.test(cls.name)) continue;
      if (this.classExtendsReactComponent(rootNode, cls)) {
        // 类组件以 classes 形式存在，在 classes 上也添加 tags
        (cls as any).tags = ['component', 'class-component'];
      }
    }

    // 标记组件导入
    for (const imp of result.imports) {
      const componentSpecifiers = imp.specifiers.filter(s => /^[A-Z][a-zA-Z0-9]*$/.test(s));
      if (componentSpecifiers.length > 0 && componentSpecifiers.length === imp.specifiers.length) {
        imp.importKind = 'component';
      }
    }
  }

  /**
   * 识别 JSX 组件组合。
   *
   * 规则: JSX 中 <PascalCase /> 标签
   * 交叉验证: 仅当 PascalCase 名称在当前文件 imports 中存在时生成 contains 边
   * 排除: HTML 标准标签（div, span, header 等）不生成边
   */
  private identifyJsxComposition(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    // 由 extractCallGraph 中的 addContainsEdges 处理
    // 此方法仅标记组件的 contains 关系（在 result 层面不直接体现，
    // 通过 CallGraphEntry.relationType = 'contains' 体现）
  }

  /**
   * 识别 Context 关系。
   *
   * 规则:
   * 1. createContext() 调用 → context-definition 标签
   * 2. useContext(ContextName) 调用 → context-consumer 标签 + depends_on 边
   */
  private identifyContextRelations(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    // 1. 遍历 AST 查找 createContext 调用
    const contextDefinitions = this.findCreateContextCalls(rootNode);

    // 2. 遍历 AST 查找 useContext 调用
    const contextConsumers = this.findUseContextCalls(rootNode);

    // 3. 标记函数
    for (const fn of result.functions) {
      // 检查函数体内是否包含 createContext
      for (const ctx of contextDefinitions) {
        if (ctx.lineNumber >= fn.lineRange[0] && ctx.lineNumber <= fn.lineRange[1]) {
          fn.tags = [...(fn.tags ?? []), 'context-definition'];
        }
      }
      // 检查函数体内是否包含 useContext
      for (const consumer of contextConsumers) {
        if (consumer.lineNumber >= fn.lineRange[0] && consumer.lineNumber <= fn.lineRange[1]) {
          fn.tags = [...(fn.tags ?? []), 'context-consumer'];
        }
      }
    }

    // 4. 标记 Context 相关导入
    const contextNames = new Set(contextDefinitions.map(c => c.name));
    for (const imp of result.imports) {
      const contextSpecifiers = imp.specifiers.filter(s => contextNames.has(s) || /Context$/.test(s));
      if (contextSpecifiers.length > 0 && contextSpecifiers.length === imp.specifiers.length) {
        imp.importKind = 'context';
      }
    }
  }

  /**
   * 识别 HOC 包装模式。
   *
   * 规则:
   * 1. export default hocName(Component) → hoc-wrapped 标签
   * 2. const Wrapped = memo(Component) → hoc-wrapped 标签
   * 3. const Wrapped = forwardRef((props, ref) => {}) → hoc-wrapped + forward-ref 标签
   * 4. connect()() 双重调用模式 → hoc-wrapped 标签
   */
  private identifyHocPatterns(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    // 遍历 AST 查找 HOC 模式
    const hocWrappers = this.findHocPatterns(rootNode);

    for (const hoc of hocWrappers) {
      // 在 exports 中查找被 HOC 包装的组件
      for (const exp of result.exports) {
        if (exp.lineNumber === hoc.lineNumber) {
          // 标记对应的函数
          for (const fn of result.functions) {
            if (fn.name === hoc.wrappedName) {
              fn.tags = [...(fn.tags ?? []), 'hoc-wrapped'];
            }
          }
        }
      }

      // 在变量声明中查找被 HOC 包装的组件
      for (const fn of result.functions) {
        if (fn.name === hoc.wrappedName && hoc.lineNumber >= fn.lineRange[0] && hoc.lineNumber <= fn.lineRange[1]) {
          fn.tags = [...(fn.tags ?? []), 'hoc-wrapped'];
        }
      }
    }

    // 标记 HOC 相关导入
    const hocNames = new Set(hocWrappers.map(h => h.hocName));
    for (const imp of result.imports) {
      const hocSpecifiers = imp.specifiers.filter(s => hocNames.has(s) || REACT_HOCS.has(s));
      if (hocSpecifiers.length > 0 && hocSpecifiers.length === imp.specifiers.length) {
        imp.importKind = 'hoc';
      }
    }
  }

  // ---- CallGraph 关系边扩展 ----

  /**
   * 添加 contains 边（父组件 → 子组件）
   */
  private addContainsEdges(entries: CallGraphEntry[], rootNode: TreeSitterNode): void {
    // 遍历 AST 查找 JSX 中 <PascalCase /> 标签
    // 交叉验证: 仅当 PascalCase 名称在 imports 中存在时生成边
    const jsxComponents = this.findJsxComponentRefs(rootNode);
    for (const ref of jsxComponents) {
      entries.push({
        caller: ref.parentFunction,
        callee: ref.componentName,
        lineNumber: ref.lineNumber,
        relationType: 'contains',
      });
    }
  }

  /**
   * 添加 depends_on 边（Context 消费 → 定义、HOC 包装）
   */
  private addDependsOnEdges(entries: CallGraphEntry[], rootNode: TreeSitterNode): void {
    // Context depends_on 边
    const contextConsumers = this.findUseContextCalls(rootNode);
    for (const consumer of contextConsumers) {
      entries.push({
        caller: consumer.parentFunction,
        callee: consumer.contextName,
        lineNumber: consumer.lineNumber,
        relationType: 'depends_on',
      });
    }

    // HOC depends_on 边
    const hocWrappers = this.findHocPatterns(rootNode);
    for (const hoc of hocWrappers) {
      entries.push({
        caller: hoc.wrappedName,
        callee: hoc.hocName,
        lineNumber: hoc.lineNumber,
        relationType: 'depends_on',
      });
    }
  }

  // ---- 辅助方法 ----

  /** 收集从 'react' 导入的所有 specifiers */
  private collectReactImports(result: StructuralAnalysis): Set<string> {
    const specifiers = new Set<string>();
    for (const imp of result.imports) {
      if (imp.source === 'react') {
        for (const spec of imp.specifiers) {
          specifiers.add(spec);
        }
      }
    }
    return specifiers;
  }

  /** 检查函数是否返回 JSX */
  private functionReturnsJsx(rootNode: TreeSitterNode, fn: StructuralAnalysis['functions'][0]): boolean {
    let found = false;
    traverse(rootNode, (node) => {
      if (found) return;
      // 定位函数节点
      if (
        (node.type === 'function_declaration' || node.type === 'arrow_function') &&
        node.startPosition.row + 1 === fn.lineRange[0]
      ) {
        // 在函数体内查找 JSX 节点
        traverse(node, (child) => {
          if (
            child.type === 'jsx_element' ||
            child.type === 'jsx_self_closing_element' ||
            child.type === 'jsx_fragment'
          ) {
            found = true;
          }
        });
      }
    });
    return found;
  }

  /** 检查函数是否有 React.FC 类型标注 */
  private hasReactFcAnnotation(rootNode: TreeSitterNode, fn: StructuralAnalysis['functions'][0]): boolean {
    let found = false;
    traverse(rootNode, (node) => {
      if (found) return;
      if (
        node.type === 'variable_declarator' &&
        node.startPosition.row + 1 === fn.lineRange[0]
      ) {
        const typeAnnotation = findChild(node, 'type_annotation');
        if (typeAnnotation) {
          const text = typeAnnotation.text;
          if (/React\.(FC|FunctionComponent|SFC)/.test(text)) {
            found = true;
          }
        }
      }
    });
    return found;
  }

  /** 识别 forwardRef 包裹的组件 */
  private identifyForwardRefComponents(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    traverse(rootNode, (node) => {
      if (node.type === 'call_expression' && node.childForFieldName('function')?.text === 'forwardRef') {
        const parent = node.parent;
        if (parent?.type === 'variable_declarator') {
          const nameNode = parent.childForFieldName('name');
          if (nameNode) {
            for (const fn of result.functions) {
              if (fn.name === nameNode.text) {
                fn.tags = [...(fn.tags ?? []), 'component', 'function-component', 'forward-ref'];
              }
            }
          }
        }
      }
    });
  }

  /** 检查类是否继承 React.Component */
  private classExtendsReactComponent(rootNode: TreeSitterNode, cls: StructuralAnalysis['classes'][0]): boolean {
    let found = false;
    traverse(rootNode, (node) => {
      if (found) return;
      if (node.type === 'class_declaration' && node.startPosition.row + 1 === cls.lineRange[0]) {
        const heritage = findChild(node, 'class_heritage');
        if (heritage) {
          const text = heritage.text;
          if (/React\.(Component|PureComponent)|extends\s+(Component|PureComponent)/.test(text)) {
            found = true;
          }
        }
      }
    });
    return found;
  }

  /** 查找 createContext 调用 */
  private findCreateContextCalls(rootNode: TreeSitterNode): Array<{ name: string; lineNumber: number }> {
    const results: Array<{ name: string; lineNumber: number }> = [];
    traverse(rootNode, (node) => {
      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (func?.text === 'createContext' || func?.text === 'React.createContext') {
          const parent = node.parent;
          if (parent?.type === 'variable_declarator') {
            const nameNode = parent.childForFieldName('name');
            if (nameNode) {
              results.push({
                name: nameNode.text,
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
        }
      }
    });
    return results;
  }

  /** 查找 useContext 调用 */
  private findUseContextCalls(rootNode: TreeSitterNode): Array<{ contextName: string; parentFunction: string; lineNumber: number }> {
    const results: Array<{ contextName: string; parentFunction: string; lineNumber: number }> = [];
    const functionStack: string[] = [];

    const walk = (node: TreeSitterNode) => {
      const isFunctionLike =
        node.type === 'function_declaration' ||
        node.type === 'arrow_function' ||
        node.type === 'function_expression';

      let pushedName = '';
      if (isFunctionLike) {
        let name: string | undefined;
        if (node.type === 'function_declaration') {
          name = (node.childForFieldName('name') ?? node.children.find(c => c.type === 'identifier'))?.text;
        } else {
          const parent = node.parent;
          if (parent?.type === 'variable_declarator') {
            name = parent.childForFieldName('name')?.text;
          }
        }
        if (name) {
          functionStack.push(name);
          pushedName = name;
        }
      }

      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (func?.text === 'useContext' || func?.text === 'React.useContext') {
          const args = node.childForFieldName('arguments');
          if (args) {
            const firstArg = args.children.find(c => c.type === 'identifier');
            if (firstArg) {
              results.push({
                contextName: firstArg.text,
                parentFunction: functionStack[functionStack.length - 1] ?? '<module>',
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
        }
      }

      for (let i = 0; i < node.childCount; i++) {
        const child = node.child(i);
        if (child) walk(child);
      }

      if (pushedName) functionStack.pop();
    };

    walk(rootNode);
    return results;
  }

  /** 查找 HOC 包装模式 */
  private findHocPatterns(rootNode: TreeSitterNode): Array<{ hocName: string; wrappedName: string; lineNumber: number }> {
    const results: Array<{ hocName: string; wrappedName: string; lineNumber: number }> = [];

    traverse(rootNode, (node) => {
      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (!func) return;

        const funcText = func.text;

        // 模式 1: memo(Component) / forwardRef(...)
        if (REACT_HOCS.has(funcText)) {
          const args = node.childForFieldName('arguments');
          if (args) {
            // forwardRef 内部是箭头函数，不提取 wrappedName
            // memo(Component) 提取 Component 名
            const firstArg = args.children.find(c => c.type === 'identifier');
            if (firstArg && /^[A-Z]/.test(firstArg.text)) {
              results.push({
                hocName: funcText,
                wrappedName: firstArg.text,
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
        }

        // 模式 2: connect()() / withRouter(Component) — 任意 HOC
        if (/^[a-z]/.test(funcText) && !REACT_HOOKS.has(funcText)) {
          const args = node.childForFieldName('arguments');
          if (args) {
            const firstArg = args.children.find(c => c.type === 'identifier' && /^[A-Z]/.test(c.text));
            if (firstArg) {
              results.push({
                hocName: funcText,
                wrappedName: firstArg.text,
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
        }

        // 模式 3: connect(mapStateToProps)(Component) — 双重调用
        if (node.type === 'call_expression' && func.type === 'call_expression') {
          const outerArgs = node.childForFieldName('arguments');
          if (outerArgs) {
            const component = outerArgs.children.find(c => c.type === 'identifier' && /^[A-Z]/.test(c.text));
            if (component) {
              const innerFunc = func.childForFieldName('function');
              if (innerFunc) {
                results.push({
                  hocName: innerFunc.text,
                  wrappedName: component.text,
                  lineNumber: node.startPosition.row + 1,
                });
              }
            }
          }
        }
      }
    });

    return results;
  }

  /** 查找 JSX 中的组件引用 */
  private findJsxComponentRefs(rootNode: TreeSitterNode): Array<{ componentName: string; parentFunction: string; lineNumber: number }> {
    const results: Array<{ componentName: string; parentFunction: string; lineNumber: number }> = [];
    const functionStack: string[] = [];

    const walk = (node: TreeSitterNode) => {
      const isFunctionLike =
        node.type === 'function_declaration' ||
        node.type === 'arrow_function' ||
        node.type === 'function_expression' ||
        node.type === 'method_definition';

      let pushedName = '';
      if (isFunctionLike) {
        let name: string | undefined;
        if (node.type === 'function_declaration') {
          name = (node.childForFieldName('name') ?? node.children.find(c => c.type === 'identifier'))?.text;
        } else if (node.type === 'method_definition') {
          name = node.children.find(c => c.type === 'property_identifier')?.text;
        } else {
          const parent = node.parent;
          if (parent?.type === 'variable_declarator') {
            name = parent.childForFieldName('name')?.text;
          }
        }
        if (name) {
          functionStack.push(name);
          pushedName = name;
        }
      }

      // 检测 JSX 元素
      if (node.type === 'jsx_self_closing_element' || node.type === 'jsx_opening_element') {
        const nameNode = node.childForFieldName('name');
        if (nameNode && /^[A-Z]/.test(nameNode.text)) {
          results.push({
            componentName: nameNode.text,
            parentFunction: functionStack[functionStack.length - 1] ?? '<module>',
            lineNumber: node.startPosition.row + 1,
          });
        }
      }

      for (let i = 0; i < node.childCount; i++) {
        const child = node.child(i);
        if (child) walk(child);
      }

      if (pushedName) functionStack.pop();
    };

    walk(rootNode);
    return results;
  }
}
```

### 4.2 识别规则交叉验证策略

为避免误判，ReactExtractor 采用多维度交叉验证：

| 模式 | 主规则 | 交叉验证 | 误判防护 |
|---|---|---|---|
| 内置 hook | 函数名 ∈ REACT_HOOKS | 必须从 `'react'` 导入 | 防止同名普通函数被误标记 |
| 自定义 hook | 函数名匹配 `use[A-Z]` | 无需额外验证（命名约定即规则） | `use[A-Z]` 是社区强约定，误判概率极低 |
| 函数组件 | 函数名 PascalCase | 返回 JSX 或 React.FC 类型标注 | 防止 PascalCase 工具函数被误标记 |
| 类组件 | 类名 PascalCase | extends React.Component / PureComponent | 类继承是确定性判断 |
| JSX 组合 | `<PascalCase />` | 名称在 imports 中存在 | 防止 HTML 自定义标签被误判 |
| Context 定义 | `createContext()` | 无需额外验证 | `createContext` 是确定性 API |
| Context 消费 | `useContext()` | 无需额外验证 | `useContext` 是确定性 API |
| HOC 包装 | 函数调用包裹 PascalCase | 排除 REACT_HOOKS + 排除内置函数 | 防止普通函数调用被误判 |

### 4.3 组件与 hook 互斥规则

一个函数**不会**同时被标记为 `component` 和 `hook`：

```
if (fn.tags?.includes('hook')) continue; // 跳过组件识别
```

**理由**：React 社区约定 hook 以 `use` 开头，组件以 PascalCase 命名。虽然 `use[A-Z]` 也是 PascalCase（如 `useCustomHook`），但 hook 命名优先级高于组件，因为 hook 的语义更具体。

**兼容性说明**：
- `ReactExtractor` 继承 `TypeScriptExtractor`，`languageIds` 相同（`['typescript', 'javascript']`）
- `ReactExtractor.extractStructure()` 先调用 `super.extractStructure()`，再增量分析，基础提取结果与原 `TypeScriptExtractor` 完全一致
- `ReactExtractor.extractCallGraph()` 先调用 `super.extractCallGraph()`，再增量添加关系边，原有调用图条目不受影响
- `TypeScriptExtractor` 仍然导出，供其他代码直接引用（如 VueSfcPlugin 内部使用）

**对 Vue/Svelte 脚本分析的影响**：

VueSfcPlugin 和 SveltePlugin 内部委托给 `tsPlugin`（即 `TreeSitterPlugin` 实例），而 `TreeSitterPlugin` 使用 `builtinExtractors` 中注册的提取器来分析 script 内容。当 `ReactExtractor` 替换 `TypeScriptExtractor` 注册后，Vue/Svelte 文件中的 `<script>` 块也将由 `ReactExtractor` 处理。

这意味着：
1. Vue/Svelte script 中的 React 特有模式（如 `useState`、`useEffect`、PascalCase 函数返回 JSX）将被标记 `tags`（如 `hook`、`component`）
2. Vue/Svelte script 中的导入将被标记 `importKind`（如 `hook`、`component`）
3. 这种行为是**可接受的**，因为：a) Vue/Svelte 项目中极少出现 React 模式；b) 即使出现，`tags` 和 `importKind` 为可选字段，不影响基础结构提取结果；c) Vue 编译宏（`defineProps`、`defineEmits`）不在 React 识别规则中，不会被误标记

## 6. Layer 3: LLM 指南

### 6.1 文件: `skills/understand/languages/react-jsx.md`

```markdown
# React JSX Language Prompt Snippet

## Key Concepts

- **Components**: PascalCase functions/classes that return JSX; function components (FC) and class components
- **Hooks**: Functions prefixed with `use` that manage state and side effects; built-in hooks (useState, useEffect, etc.) and custom hooks
- **JSX Composition**: Components rendered within other components via `<PascalCase />` syntax
- **Context**: `createContext` for defining shared state; `useContext` for consuming; Provider/Consumer pattern
- **HOC (Higher-Order Components)**: Functions that wrap components (memo, forwardRef, connect, withRouter)
- **forwardRef**: Exposes a ref to parent components; used for imperative handles and DOM access
- **React.memo**: Performance optimization via shallow prop comparison
- **Custom Hooks**: `use[A-Z]` named functions that encapsulate reusable stateful logic
- **Render Props**: Pattern where a component receives a function as a prop to control rendering
- **Strict Mode**: Development mode wrapper for detecting side effects

## Notable File Patterns

- `*.tsx` — TypeScript React components
- `*.jsx` — JavaScript React components
- `use*.ts` — Custom hook files
- `*.test.tsx`, `*.spec.tsx` — React component test files
- `*.stories.tsx` — Storybook stories
- `context/*.tsx` — React Context definitions
- `hooks/*.ts` — Custom hooks directory

## Edge Patterns

- React function components are `contains` by the parent component that renders them via JSX
- `useState`/`useReducer` create `depends_on` edges from the component to its state
- `useContext` creates `depends_on` edges from the consumer component to the Context definition
- `createContext` creates a `context-definition` node that other components depend on
- `memo(Component)` creates `depends_on` edge from the component to the memo HOC
- `forwardRef` creates `depends_on` edge from the component to the forwardRef HOC
- Custom hooks (`useX`) create `depends_on` edges from the consuming component to the hook module
- HOC patterns (`connect()()`, `withRouter()`) create `depends_on` edges from the wrapped component to the HOC
- CSS-in-JS patterns (styled-components, emotion) create `related` edges to style definitions

## Summary Style

> "React function component implementing a search form with debounced input, uses useState for form state and useCallback for the submit handler."
> "Custom hook encapsulating authentication logic, uses useContext for AuthContext and useState for loading state."
> "React component wrapped with React.memo for performance optimization, renders a list of items with virtualized scrolling."
```
