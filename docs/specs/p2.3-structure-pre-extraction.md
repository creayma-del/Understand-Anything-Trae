## 1. 目标与范围

### 1.1 问题

当前 `/understand` 管线中，Phase 2 派遣 N 个 file-analyzer 子代理，每个子代理内部独立运行 `extract-structure.mjs`，导致：

1. **tree-sitter WASM 重复初始化 N 次**：单次初始化约 260ms（WASM 运行时 + 10 种语言语法加载），300 文件项目产生 ~20 个批次，累计 ~5s 纯初始化开销
2. **文件重复读取 + 解析**：每个子代理进程独立读取并解析本批次文件，进程间无法共享解析缓存
3. **子代理进程启动开销**：每次 `node extract-structure.mjs` 涉及 Node.js 进程启动 + ESM 模块加载，~200-500ms/次

### 1.2 目标

- tree-sitter 初始化次数从 N 降为 1（仅主流程 1 次）
- 消除子代理内的结构提取进程启动 + 文件 I/O 开销
- 结构提取结果与优化前完全等价（确定性操作，位置不影响结果）
- 300 文件项目节省 ~5s 初始化 + ~15s 重复解析

### 1.3 范围

**包含**：
- 新增 `extract-all-structure.mjs` 脚本
- SKILL.md 新增 Phase 1.7 定义
- SKILL.md Phase 2 dispatch prompt 注入结构预提取数据
- file-analyzer.md Phase 1 改为读取预提取结果
- 新增 7 个测试用例（EAS-001 ~ EAS-007）

**不包含**：
- 与 `extract-import-map.mjs` / `compute-batches.mjs` 的合并（属于优化 B，阶段 4）
- 批次参数调优（属于优化 C，阶段 1）
- imports 边确定性生成（属于优化 F，阶段 2）
- Phase 1 / Phase 3 LLM 调用确定性替代（属于优化 D/E，阶段 5）

### 1.4 约束

- 结构提取结果必须与 `extract-structure.mjs` 输出格式完全一致
- 不修改 `extract-structure.mjs` 本身（保留为独立入口，供增量更新等场景使用）
- 不修改 `@understand-anything-trae/core` 的公共 API
- 不引入新的外部依赖
- 增量更新路径必须完全兼容

## 2. 背景分析

### 2.1 现有数据流

```
Phase 1.5 → Phase 2
               │
               ├─ file-analyzer #1
               │    ├─ Step 1: 创建 input JSON (batchFiles + batchImportData)
               │    ├─ Step 2: node extract-structure.mjs <input> <output>
               │    │           → 初始化 tree-sitter
               │    │           → 遍历本批次文件
               │    │           → 输出结构数据
               │    └─ Step 3: 读取结构数据 → LLM 语义分析
               │
               ├─ file-analyzer #2
               │    └─ (同上，tree-sitter 再次初始化)
               │
               └─ ... file-analyzer #N
                    └─ (同上，tree-sitter 第 N 次初始化)
```

### 2.2 优化后数据流

```
Phase 1.5 → Phase 1.7 → Phase 2
               │            │
               │            ├─ file-analyzer #1
               │            │    └─ Step 1: 从 dispatch prompt 读取预提取结构数据
               │            │    └─ Step 2: LLM 语义分析
               │            │
               │            ├─ file-analyzer #2
               │            │    └─ (同上，无需 tree-sitter)
               │            │
               │            └─ ... file-analyzer #N
               │                 └─ (同上，无需 tree-sitter)
               │
               └─ node extract-all-structure.mjs $PROJECT_ROOT $OUTPUT_DIR
                    → 读取 scan-result.json
                    → 初始化 tree-sitter (1 次)
                    → 遍历所有文件
                    → 输出 structure-results.json
```

### 2.3 `extract-structure.mjs` 输出格式分析

当前 `extract-structure.mjs` 的输出格式（`buildResult()` 函数产出）：

```json
{
  "scriptCompleted": true,
  "filesAnalyzed": 5,
  "filesSkipped": ["path/to/binary.wasm"],
  "results": [
    {
      "path": "src/index.ts",
      "language": "typescript",
      "fileCategory": "code",
      "totalLines": 150,
      "nonEmptyLines": 120,
      "functions": [
        { "name": "main", "startLine": 10, "endLine": 45, "params": ["config", "options"] }
      ],
      "classes": [
        { "name": "App", "startLine": 50, "endLine": 140, "methods": ["init", "run"], "properties": ["config", "logger"] }
      ],
      "exports": [
        { "name": "App", "line": 50, "isDefault": false }
      ],
      "callGraph": [
        { "caller": "main", "callee": "initApp", "lineNumber": 15 }
      ],
      "metrics": {
        "importCount": 5,
        "exportCount": 3,
        "functionCount": 4,
        "classCount": 1
      }
    }
  ]
}
```

非代码文件可能额外包含：`sections`、`definitions`、`services`、`endpoints`、`steps`、`resources`。

`buildResult()` 函数签名：

```javascript
export function buildResult(file, totalLines, nonEmptyLines, analysis, callGraph, batchImportData)
```

其中 `file` 对象需要 `{ path, language, fileCategory }`，`batchImportData` 用于计算 `metrics.importCount`。

### 2.4 `scan-result.json` 格式

Phase 1 产出的 `scan-result.json` 包含：

```json
{
  "projectName": "...",
  "projectDescription": "...",
  "languages": ["typescript", "css"],
  "frameworks": ["React"],
  "files": [
    { "path": "src/index.ts", "language": "typescript", "sizeLines": 150, "fileCategory": "code" },
    { "path": "src/style.css", "language": "css", "sizeLines": 50, "fileCategory": "code" }
  ],
  "importMap": {
    "src/index.ts": ["src/utils.ts", "src/config.ts"]

```

关键字段：
- `files[]`：文件列表，每项含 `path`、`language`、`sizeLines`、`fileCategory`
- `importMap`：预解析的导入映射（`batchImportData` 的来源）

## 3. 接口设计

### 3.1 `extract-all-structure.mjs` CLI 接口

```
node extract-all-structure.mjs <projectRoot> <outputDir>
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `projectRoot` | 绝对路径 | 项目根目录 |
| `outputDir` | 绝对路径 | 中间文件输出目录（通常为 `$PROJECT_ROOT/.understand-anything-trae/intermediate`） |

**输入**：
- 读取 `<outputDir>/scan-result.json`（Phase 1 产出）

**输出**：
- 写入 `<outputDir>/structure-results.json`

**退出码**：
- `0`：成功
- `1`：失败（输入缺失、JSON 解析错误、tree-sitter 初始化失败等）

**stderr**：
- 正常运行时输出统计信息：`Files analyzed: N, Files skipped: M`
- 警告信息以 `Warning:` 前缀输出，供 SKILL.md 捕获到 `$PHASE_WARNINGS`
- 错误信息以 `extract-all-structure.mjs failed:` 前缀输出

### 3.2 `structure-results.json` 输出格式

与 `extract-structure.mjs` 输出格式完全一致：

```json
{
  "scriptCompleted": true,
  "filesAnalyzed": 300,
  "filesSkipped": [],
  "results": [
    {
      "path": "src/index.ts",
      "language": "typescript",
      "fileCategory": "code",
      "totalLines": 150,
      "nonEmptyLines": 120,
      "functions": [...],
      "classes": [...],
      "exports": [...],
      "callGraph": [...],
      "metrics": { ... }
    }
  ]
}
```

**设计决策**：输出格式与 `extract-structure.mjs` 保持一致，原因：
1. file-analyzer.md 的 Phase 2（LLM 语义分析）已适配此格式，无需修改解析逻辑
2. `buildResult()` 函数可直接复用，确保字段映射逻辑一致
3. 未来优化 B 合并脚本时，格式统一降低迁移成本

## 4. 实现细节

### 4.1 `extract-all-structure.mjs` 完整设计

```javascript
#!/usr/bin/env node
/**
 * extract-all-structure.mjs
 *
 * Phase 1.7 of /understand: pre-extract structural data for ALL project files.
 * Runs tree-sitter initialization ONCE, processes all files, and outputs
 * structure-results.json for Phase 2 subagents to consume.
 *
 * Usage:
 *   node extract-all-structure.mjs <projectRoot> <outputDir>
 *
 * Input:  <outputDir>/scan-result.json (Phase 1 output)
 * Output: <outputDir>/structure-results.json
 */

import { createRequire } from 'node:module';
import { dirname, resolve, join } from 'node:path';
import { fileURLToPath, pathToFileURL } from 'node:url';
import { existsSync, readFileSync, realpathSync, writeFileSync } from 'node:fs';

const __dirname = dirname(fileURLToPath(import.meta.url));
const pluginRoot = resolve(__dirname, '../..');
const require = createRequire(resolve(pluginRoot, 'package.json'));

// Resolve @understand-anything-trae/core (与 extract-structure.mjs 相同)
let core;
try {
  core = await import(pathToFileURL(require.resolve('@understand-anything-trae/core')).href);
} catch {
  core = await import(pathToFileURL(resolve(pluginRoot, 'packages/core/dist/index.js')).href);
}

const { TreeSitterPlugin, PluginRegistry, builtinLanguageConfigs, registerAllParsers, CssPlugin } = core;

// 复用 extract-structure.mjs 的 buildResult() 函数
// 通过动态导入获取（buildResult 是 export 的纯函数）
let buildResult;
try {
  const extractModule = await import(pathToFileURL(resolve(__dirname, 'extract-structure.mjs')).href);
  buildResult = extractModule.buildResult;
} catch {
  // 回退：内联 buildResult 的完整实现（确保即使动态导入失败也能工作）
  // ... 内联 buildResult 实现 ...
}

async function main() {
  const [,, projectRoot, outputDir] = process.argv;
  if (!projectRoot || !outputDir) {
    process.stderr.write('Usage: node extract-all-structure.mjs <projectRoot> <outputDir>\n');
    process.exit(1);
  }

  // 1. 读取 scan-result.json
  const scanResultPath = join(outputDir, 'scan-result.json');
  if (!existsSync(scanResultPath)) {
    process.stderr.write(`extract-all-structure.mjs failed: scan-result.json not found at ${scanResultPath}\n`);
    process.exit(1);
  }

  const scanResult = JSON.parse(readFileSync(scanResultPath, 'utf-8'));
  const { files: allFiles, importMap } = scanResult;

  if (!Array.isArray(allFiles) || allFiles.length === 0) {
    process.stderr.write('Warning: extract-all-structure: no files found in scan-result.json\n');
    // 写入空结果
    const emptyOutput = { scriptCompleted: true, filesAnalyzed: 0, filesSkipped: [], results: [] };
    writeFileSync(join(outputDir, 'structure-results.json'), JSON.stringify(emptyOutput, null, 2), 'utf-8');
    process.exit(0);
  }

  // 2. 初始化 tree-sitter（一次）
  const tsConfigs = builtinLanguageConfigs.filter(c => c.treeSitter);
  const tsPlugin = new TreeSitterPlugin(tsConfigs);
  await tsPlugin.init();

  const registry = new PluginRegistry();
  registry.register(tsPlugin);
  const cssPlugin = new CssPlugin();
  registerAllParsers(registry, tsPlugin, cssPlugin);

  // 3. 遍历所有文件
  const results = [];
  const filesSkipped = [];

  for (const file of allFiles) {
    const absolutePath = join(projectRoot, file.path);

    // 读取文件内容
    let content;
    try {
      content = readFileSync(absolutePath, 'utf-8');
    } catch {
      filesSkipped.push(file.path);
      continue;
    }

    // 行数统计（与 extract-structure.mjs 逻辑一致）
    const lines = content.split('\n');
    const totalLines = content.endsWith('\n') ? Math.max(0, lines.length - 1) : lines.length;
    const nonEmptyLines = lines.filter(l => l.trim().length > 0).length;

    // 结构分析
    let analysis = null;
    try {
      analysis = registry.analyzeFile(file.path, content);
    } catch {
      // 分析失败，降级为基础指标
    }

    // 调用图提取（仅代码/脚本文件）
    let callGraph = null;
    if (file.fileCategory === 'code' || file.fileCategory === 'script') {
      try {
        const cg = registry.extractCallGraph(file.path, content);
        if (cg && cg.length > 0) {
          callGraph = cg.map(entry => ({
            caller: entry.caller,
            callee: entry.callee,
            lineNumber: entry.lineNumber,
          }));
        }
      } catch {
        // 调用图提取失败，非致命
      }
    }

    // 构建结果（复用 buildResult）
    const batchImportData = importMap || {};
    const result = buildResult(file, totalLines, nonEmptyLines, analysis, callGraph, batchImportData);
    results.push(result);
  }

  // 4. 写入输出
  const output = {
    scriptCompleted: true,
    filesAnalyzed: results.length,
    filesSkipped,
    results,
  };

  const outputPath = join(outputDir, 'structure-results.json');
  writeFileSync(outputPath, JSON.stringify(output, null, 2), 'utf-8');

  if (!existsSync(outputPath)) {
    throw new Error(`output file missing after write: ${outputPath}`);
  }

  process.stderr.write(`Files analyzed: ${results.length}, Files skipped: ${filesSkipped.length}\n`);
}

// CLI 入口守卫（与 extract-structure.mjs 相同的 symlink 处理）
function isCliEntry() {
  if (!process.argv[1]) return false;
  try {
    const modulePath = realpathSync(fileURLToPath(import.meta.url));
    const argvPath = realpathSync(process.argv[1]);
    return modulePath === argvPath;
  } catch {
    return false;
  }
}

if (isCliEntry()) {
  try {
    await main();
  } catch (err) {
    process.stderr.write(`extract-all-structure.mjs failed: ${err.message}\n${err.stack}\n`);
    process.exit(1);
  }
}
```

### 4.2 关键实现决策

#### 4.2.1 复用 `buildResult()` 的方式

**方案**：通过动态 `import()` 从 `extract-structure.mjs` 导入 `buildResult` 函数。

**理由**：
- `buildResult` 是纯函数（无 I/O、无副作用），适合复用
- 避免代码重复，确保字段映射逻辑始终一致
- `extract-structure.mjs` 已将 `buildResult` 导出（`export function buildResult`）
- 如果动态导入失败（如路径问题），回退到内联实现

**备选方案**：将 `buildResult` 提取到独立模块（如 `shared/build-result.mjs`）。但当前阶段不修改 `extract-structure.mjs`（约束条件），因此采用动态导入。

#### 4.2.2 `batchImportData` 来源

`buildResult()` 的第 6 个参数 `batchImportData` 用于计算 `metrics.importCount`。在 `extract-structure.mjs` 中，此数据来自子代理的输入 JSON；在 `extract-all-structure.mjs` 中，此数据来自 `scan-result.json` 的 `importMap` 字段。

**映射关系**：
- `scan-result.json.importMap` 的格式：`{ "src/index.ts": ["src/utils.ts", "src/config.ts"] }`
- `buildResult()` 中 `batchImportData` 的使用方式：`batchImportData?.[file.path]` → 返回导入路径数组
- 两者格式一致，可直接传递 `importMap` 作为 `batchImportData`

#### 4.2.3 错误处理策略

| 场景 | 处理方式 |
|------|---------|
| `scan-result.json` 不存在 | 退出码 1，stderr 输出错误信息 |
| `scan-result.json` JSON 解析失败 | 退出码 1，stderr 输出错误信息 |
| `scan-result.json` 中 `files` 为空数组 | 写入空结果，退出码 0，stderr 输出 Warning |
| 单个文件读取失败 | 跳过该文件，加入 `filesSkipped`，继续处理 |
| 单个文件分析失败 | `analysis = null`，`buildResult` 返回基础指标 |
| tree-sitter 初始化失败 | 退出码 1，stderr 输出错误信息 |
| 输出文件写入失败 | 退出码 1，stderr 输出错误信息 |

#### 4.2.4 确定性保证

`extract-all-structure.mjs` 的输出必须是确定性的：
- 文件遍历顺序：按 `scan-result.json` 中 `files[]` 数组的顺序
- JSON 序列化：使用 `JSON.stringify(output, null, 2)`（与 `extract-structure.mjs` 一致）
- 无时间戳、无随机数、无环境依赖

### 5.3 增量更新路径兼容性

增量更新路径中，Phase 2 的 file-analyzer 子代理同样需要结构数据。两种方案：

**方案 A**：增量更新也运行 `extract-all-structure.mjs`（仅处理变更文件）

- 优点：主流程逻辑统一
- 缺点：需要新增 `--changed-files` 参数支持，或创建变更文件的 `scan-result.json` 子集

**方案 B**：增量更新时 file-analyzer 子代理仍运行 `extract-structure.mjs`

- 优点：增量更新路径无需修改
- 缺点：两条路径不一致

**选择方案 A**：增量更新时，主流程在 Phase 2 前运行 `extract-all-structure.mjs`，但仅处理变更文件。具体实现：

1. `compute-batches.mjs --changed-files` 产出的 `batches.json` 仅包含变更文件的批次
2. 主流程在 Phase 1.7 运行 `extract-all-structure.mjs`，该脚本读取的 `scan-result.json` 包含所有文件
3. `extract-all-structure.mjs` 仍处理所有文件（因为结构数据可能被其他批次引用）
4. dispatch prompt 中筛选本批次文件的结构数据

**实际影响**：增量更新时 `extract-all-structure.mjs` 仍处理全量文件，但 tree-sitter 仅初始化 1 次（而非 N 次），性能仍有提升。后续优化 B 可进一步优化为仅处理变更文件。

### 6.4 测试辅助函数

```javascript
/**
 * 创建临时项目目录并写入文件树 + scan-result.json。
 * @param {Object} files - { 'src/index.ts': '...', ... }
 * @param {Object} options - { importMap?: {...} }
 * @returns {string} 临时项目根目录
 */
function setupProject(files, options = {}) {
  const root = mkdtempSync(join(tmpdir(), 'ua-eas-test-'));

  // 写入源文件
  for (const [relPath, contents] of Object.entries(files)) {
    const abs = join(root, relPath);
    mkdirSync(dirname(abs), { recursive: true });
    writeFileSync(abs, contents, 'utf-8');
  }

  // 生成 scan-result.json
  const scanFiles = Object.keys(files).map(path => {
    const ext = extname(path);
    const language = detectLanguage(ext);
    const fileCategory = detectCategory(ext);
    const sizeLines = files[path].split('\n').length;
    return { path, language, sizeLines, fileCategory };
  });

  const intermediateDir = join(root, '.understand-anything-trae', 'intermediate');
  mkdirSync(intermediateDir, { recursive: true });

  const scanResult = {
    projectName: 'test-project',
    projectDescription: 'Test project',
    languages: [...new Set(scanFiles.map(f => f.language))],
    frameworks: [],
    files: scanFiles,
    importMap: options.importMap || {},
  };

  writeFileSync(join(intermediateDir, 'scan-result.json'), JSON.stringify(scanResult), 'utf-8');

  return root;
}

/**
 * 运行 extract-all-structure.mjs 脚本。
 */
function runScript(projectRoot, outputDir) {
  const result = spawnSync('node', [SCRIPT, projectRoot, outputDir], { encoding: 'utf-8' });
  let output = null;
  try {
    output = JSON.parse(readFileSync(join(outputDir, 'structure-results.json'), 'utf-8'));
  } catch {
    /* output missing on hard failure */
  }
  return { status: result.status, stdout: result.stdout, stderr: result.stderr, output };
}
```

### 6.5 EAS-001 测试用例示例

```javascript
describe('extract-all-structure.mjs — TypeScript project', () => {
  let projectRoot;
  let outputDir;

  afterEach(() => {
    if (projectRoot) {
      rmSync(projectRoot, { recursive: true, force: true });
      projectRoot = null;
    }
  });

  it('EAS-001: extracts functions, classes, exports, and callGraph from TypeScript files', () => {
    projectRoot = setupProject({
      'src/index.ts': `
import { helper } from './utils';
import { AppConfig } from './config';

export function main(opts: unknown) {
  helper(opts);
  const cfg = new AppConfig();
  cfg.init();
}

export class Runner {
  constructor(private config: AppConfig) {}
  run() { this.config.validate(); }
}
`,
      'src/utils.ts': `export function helper(x: unknown) { return x; }\n`,
      'src/config.ts': `export class AppConfig { init() {} validate() {} }\n`,
    }, {
      importMap: {
        'src/index.ts': ['src/utils.ts', 'src/config.ts'],
      },
    });

    outputDir = join(projectRoot, '.understand-anything-trae', 'intermediate');
    const result = runScript(projectRoot, outputDir);

    expect(result.status).toBe(0);
    expect(result.output.scriptCompleted).toBe(true);
    expect(result.output.filesAnalyzed).toBe(3);
    expect(result.output.filesSkipped).toEqual([]);

    // index.ts 应包含函数和类
    const indexResult = result.output.results.find(r => r.path === 'src/index.ts');
    expect(indexResult).toBeDefined();
    expect(indexResult.functions.length).toBeGreaterThan(0);
    expect(indexResult.classes.length).toBeGreaterThan(0);
    expect(indexResult.exports.length).toBeGreaterThan(0);
    expect(indexResult.callGraph.length).toBeGreaterThan(0);
    expect(indexResult.metrics.importCount).toBe(2);
    expect(indexResult.metrics.functionCount).toBeGreaterThan(0);
    expect(indexResult.metrics.classCount).toBeGreaterThan(0);
  });
});
```

## 9. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| `buildResult()` 动态导入失败 | 低 | 高 | 内联 `buildResult` 实现作为回退；测试覆盖动态导入路径 |
| `structure-results.json` 过大导致 dispatch prompt 超长 | 中 | 中 | 300 文件项目的结构数据约 200-500KB（JSON），远在 LLM 上下文窗口内；如需优化可按批次筛选后注入（当前设计已按批次筛选） |
| `scan-result.json` 缺少 `importMap` 字段 | 低 | 中 | `buildResult()` 中 `batchImportData` 为可选参数，缺失时 `metrics.importCount` 回退到 parser 自身的 imports 列表 |
| file-analyzer 子代理仍尝试运行 `extract-structure.mjs` | 低 | 低 | dispatch prompt 中明确标注 "do NOT re-run extract-structure.mjs"；即使运行也不影响正确性（仅浪费资源） |
| 增量更新时 `extract-all-structure.mjs` 处理全量文件 | 低 | 低 | 增量更新场景文件数通常较少，全量提取开销可接受；后续优化 B 可进一步优化 |
| `extract-all-structure.mjs` 与 `extract-structure.mjs` 输出不一致 | 低 | 高 | EAS-004 确定性测试确保输出稳定；未来优化 B 的 AP-006 测试确保结果等价 |
