# P1.2: HTML Parser 详细设计规格

> 版本: 1.0
> 日期: 2026-06-05
> 状态: 已评审（修复 HIGH 问题后）
> 变更: 新增 HTML 文件结构化分析能力 — 使用 node-html-parser 解析 HTML DOM，提取元素结构、script/link 引用、语义标签与内联内容，实现确定性结构化分析

## 1. 目标与范围

为项目新增 HTML 文件（`.html` / `.htm`）的结构化分析能力，包括：

- **语言配置升级**：将现有 `htmlConfig` 从纯非代码语言提升为可结构化分析的语言
- **确定性结构化提取**：使用 `node-html-parser` 解析 HTML DOM，提取元素、属性、引用关系
- **HtmlPlugin 实现**：独立 AnalyzerPlugin，提供 `analyzeFile()` / `resolveImports()` 能力
- **导入解析增强**：`extract-import-map.mjs` 支持 HTML 文件的 `<script src>` / `<link href>` 解析
- **类型系统扩展**：`StructuralAnalysis` 新增 `htmlElements` 字段

### 1.1 不在范围内

- HTML 文件的 call graph 提取（HTML 无可调用函数）
- HTML 文件的 reference 提取（暂不实现 `extractReferences`）
- CSS 内联样式的深度解析
- JavaScript 内联 `<script>` 内容的语法分析（仅标记存在性，不解析 JS 逻辑）
- HTML 模板语言支持（Handlebars、EJS、Pug 等）

## 2. HTML 文件结构分析

HTML 文件是 Web 项目的入口和骨架，典型结构如下：

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>应用标题</title>
  <link rel="stylesheet" href="./styles/main.css" />
  <script src="./scripts/app.js" defer></script>
</head>
<body>
  <header>
    <nav aria-label="主导航">
      <a href="/home">首页</a>
      <a href="/about">关于</a>
    </nav>
  </header>
  <main>
    <article>
      <h1>文章标题</h1>
      <p>正文内容</p>
    </article>
  </main>
  <footer>
    <p>&copy; 2026</p>
  </footer>
  <script>
    // 内联脚本
    console.log('页面已加载');
  </script>
</body>
</html>
```

### 2.1 提取策略：node-html-parser

**选型理由**：HTML 文件不需要 tree-sitter 解析（无 tree-sitter-html 可用 WASM），使用轻量级纯 JS 解析器 `node-html-parser` 即可完成确定性 DOM 遍历。

```
阶段 1: node-html-parser.parse() 解析 HTML 文件
  → 获取 DOM 树：RootElement / 子节点
  → 遍历 DOM，提取所有元素节点
  → 记录标签名、属性、行范围、自闭合状态

阶段 2: 从 DOM 中提取结构化信息
  → <script src="..."> → 识别为外部脚本引用（imports）
  → <link rel="stylesheet" href="..."> → 识别为外部样式表引用（imports）
  → <link rel="modulepreload" href="..."> → 识别为模块预加载引用（imports）
  → <link rel="icon|prefetch|preconnect|..." href="..."> → 识别为关联资源（related），不纳入 imports
  → 语义标签（header/nav/main/article/section/footer）→ 识别为 sections
  → 内联 <script> / <style> → 记录存在性与行范围
  → <meta> 标签 → 提取关键元信息（charset、viewport、description）
  → <form> 标签 → 提取 action / method 属性
```

### 2.2 与 TreeSitterPlugin 的关系

HTML 不使用 TreeSitterPlugin，原因：

1. `htmlConfig` 不包含 `treeSitter` 字段（无可用 WASM grammar）
2. HTML 是声明式标记语言，不需要语法级别的函数/类提取
3. `node-html-parser` 足以完成 DOM 遍历与属性提取

### 4.4 analyzeFile() 完整实现逻辑

```typescript
analyzeFile(filePath: string, content: string): StructuralAnalysis {
  // [降级守卫] 解析失败时返回空分析
  let root: HTMLElement;
  try {
    root = parseHtml(content, {
      comment: false,       // 不保留注释节点
      blockTextElements: {  // 块级文本元素，保留内容
        script: true,
        style: true,
        pre: true,
      },
    });
  } catch {
    return {
      functions: [],
      classes: [],
      imports: [],
      exports: [],
    };
  }

  // 提取所有元素节点
  const htmlElements: HtmlElementInfo[] = [];
  const imports: StructuralAnalysis["imports"] = [];
  const sections: SectionInfo[] = [];

  // 语义标签集合
  const SEMANTIC_TAGS = new Set([
    "header", "nav", "main", "article", "section",
    "aside", "footer", "details", "summary", "figure",
  ]);

  // 可导入的 <link> rel 值集合
  // 仅 rel="stylesheet" / rel="modulepreload" 纳入 imports，
  // 其他 rel（icon、prefetch、preconnect 等）为关联资源，不纳入 imports
  const IMPORTABLE_LINK_RELS = new Set(["stylesheet", "modulepreload"]);

  // 外部引用标签映射：标签 → 引用属性
  const REF_ATTR_MAP: Record<string, string> = {
    script: "src",
    link: "href",
    img: "src",
    source: "src",
    iframe: "src",
  };

  // 预计算行号映射表（一次构建，O(1) 查找）
  const lineMap = buildLineMap(content);

  // DOM 遍历
  const visit = (node: HTMLElement) => {
    // 仅处理元素节点（node.nodeType === 1）
    if (node.nodeType !== 1) return;

    const tag = node.tagName.toLowerCase();
    const attrs: Record<string, string> = {};
    for (const [key, value] of Object.entries(node.attributes)) {
      attrs[key] = value ?? "";
    }

    // 通过预计算行号映射表计算行范围（含 range[-1,-1] 防御）
    const lineRange = computeLineRange(content, node, lineMap);

    htmlElements.push({
      tag,
      lineRange,
      attributes: attrs,
      isSelfClosing: isVoidElement(tag),
    });

    // 提取外部引用 → imports
    const refAttr = REF_ATTR_MAP[tag];
    if (refAttr && attrs[refAttr]) {
      const source = attrs[refAttr];

      // <link> 标签：仅 rel="stylesheet" / rel="modulepreload" 纳入 imports
      if (tag === "link") {
        const relValues = (attrs.rel ?? "").toLowerCase().split(/\s+/);
        const importableRel = relValues.find(r => IMPORTABLE_LINK_RELS.has(r));
        if (importableRel && !isExternalUrl(source)) {
          imports.push({
            source,
            specifiers: [importableRel], // "stylesheet" 或 "modulepreload"
            lineNumber: lineRange[0],
          });
        }
        // 其他 rel（icon、prefetch、preconnect 等）不纳入 imports
      } else if (!isExternalUrl(source)) {
        imports.push({
          source,
          specifiers: [tag === "script" ? "script" : "resource"],
          lineNumber: lineRange[0],
        });
      }
    }

    // 提取语义标签 → sections
    if (SEMANTIC_TAGS.has(tag)) {
      sections.push({
        name: attrs.id ?? attrs["aria-label"] ?? tag,
        level: semanticTagLevel(tag),
        lineRange,
      });
    }

    // 递归子节点
    for (const child of node.childNodes) {
      if (child.nodeType === 1) {
        visit(child as HTMLElement);
      }
    }
  };

  visit(root);

  return {
    functions: [],
    classes: [],
    imports,
    exports: [],
    htmlElements,
    sections: sections.length > 0 ? sections : undefined,
  };
}
```

### 4.5 行号计算辅助函数

`node-html-parser` 不直接提供行号信息，需通过元素在源文本中的偏移量计算。

**性能关键**：行号映射表必须在 `analyzeFile()` 中仅构建一次（`buildLineMap`），然后传入 `computeLineRange` 复用，避免每个元素重复构建导致 O(n²) 开销。

**防御性关键**：当 `range` 选项未设置或解析失败时，`node.range` 返回 `[-1,-1]`（`enumerable: false`），必须检测此情况并回退到从源文本搜索标签位置计算行号。

```typescript
/**
 * 预计算源文本的行号映射表。
 * 返回 Uint32Array，其中 lineMap[offset] = 行号（从 1 开始）。
 * 一次构建 O(n)，后续每个偏移量查找 O(1)。
 */
function buildLineMap(content: string): Uint32Array {
  const lineMap = new Uint32Array(content.length);
  let line = 1;
  for (let i = 0; i < content.length; i++) {
    lineMap[i] = line;
    if (content[i] === "\n") line++;
  }
  return lineMap;
}

/**
 * 计算元素在文件中的行范围。
 * node-html-parser 的 HTMLElement 有 range: [startOffset, endOffset]。
 *
 * 防御性检查：当 range 未设置或解析失败时，node.range 返回 [-1,-1]
 * （enumerable: false），此时回退到从源文本搜索标签位置计算行号。
 */
function computeLineRange(
  content: string,
  node: HTMLElement,
  lineMap: Uint32Array,
): [number, number] {
  const [start, end] = node.range;

  // 防御性检查：range 为 [-1,-1] 时回退到逐行计数
  if (start === -1 || end === -1) {
    return fallbackLineRange(content, node);
  }

  // O(1) 行号查找
  const startLine = lineMap[start] ?? 1;
  const endLine = lineMap[Math.max(0, end - 1)] ?? startLine; // -1 因为 range[1] 是 exclusive
  return [startLine, endLine];
}

/**
 * range 不可用时的降级行号计算：从源文本中搜索元素标签位置，
 * 逐字符计数换行符确定行号。
 */
function fallbackLineRange(content: string, node: HTMLElement): [number, number] {
  const tag = node.tagName.toLowerCase();
  const tagPattern = `<${tag}`;
  const idx = content.indexOf(tagPattern);
  if (idx === -1) return [1, 1];
  // 逐字符计数换行符
  let line = 1;
  for (let i = 0; i < idx; i++) {
    if (content[i] === "\n") line++;
  }
  return [line, line];
}
```

### 4.6 辅助工具函数

```typescript
/** HTML void 元素（自闭合标签）集合 */
const VOID_ELEMENTS = new Set([
  "area", "base", "br", "col", "embed", "hr", "img",
  "input", "link", "meta", "param", "source", "track", "wbr",
]);

function isVoidElement(tag: string): boolean {
  return VOID_ELEMENTS.has(tag);
}

/** 判断是否为外部 URL（不需要解析为项目内部引用） */
function isExternalUrl(url: string): boolean {
  return /^(https?:)?\/\//i.test(url) ||
         /^(data:|blob:|mailto:|tel:|javascript:)/i.test(url);
}

/** 语义标签的层级映射（用于 SectionInfo.level） */
function semanticTagLevel(tag: string): number {
  const LEVEL_MAP: Record<string, number> = {
    header: 1,
    nav: 1,
    main: 1,
    footer: 1,
    article: 2,
    section: 2,
    aside: 2,
    details: 3,
    summary: 4,
    figure: 3,
  };
  return LEVEL_MAP[tag] ?? 2;
}
```

### 4.7 resolveImports() 实现

```typescript
resolveImports(filePath: string, content: string): ImportResolution[] {
  const analysis = this.analyzeFile(filePath, content);
  const dir = filePath.includes("/")
    ? filePath.slice(0, filePath.lastIndexOf("/"))
    : "";

  return analysis.imports
    .filter(imp => !isExternalUrl(imp.source))
    .map(imp => {
      const source = imp.source;
      // 解析相对路径
      let resolvedPath: string;
      if (source.startsWith("./") || source.startsWith("../")) {
        resolvedPath = resolveRelativePath(dir, source);
      } else if (source.startsWith("/")) {
        // 绝对路径（项目根目录）
        resolvedPath = source.slice(1); // 去掉前导 /
      } else {
        resolvedPath = dir ? `${dir}/${source}` : source;
      }

      return {
        source,
        resolvedPath,
        specifiers: imp.specifiers,
      };
    });
}
```

### 4.8 降级处理

HtmlPlugin 的降级策略：

| 场景 | 处理方式 |
|---|---|
| HTML 解析失败（malformed HTML） | 返回空 StructuralAnalysis（functions/classes/imports/exports 均为空数组） |
| node-html-parser 未安装 | 插件注册时检查，未安装则不注册 HtmlPlugin |
| 空 HTML 文件 | 返回空 StructuralAnalysis |
| 仅含 DOCTYPE 的 HTML | 返回空 StructuralAnalysis（DOCTYPE 不产生元素节点） |
| 包含外部 URL 引用 | 跳过，不纳入 imports（无法解析为项目内部路径） |
| 包含 data: URI | 跳过，不纳入 imports |
| `node.range` 返回 [-1,-1] | `computeLineRange` 检测到 `start === -1 \|\| end === -1` 时，回退到 `fallbackLineRange` 从源文本搜索标签位置计算行号 |
| `<link rel="icon">` 等非导入型 link | 不纳入 imports，仅记录在 `htmlElements` 中 |

## 6. 导入解析增强

### 6.1 文件: `skills/understand/extract-import-map.mjs`

#### 6.1.1 HTML 语言识别

HtmlPlugin 注册后，`registry.analyzeFile()` 对 `.html` 文件返回包含 `imports` 的 `StructuralAnalysis`。但 `extract-import-map.mjs` 需要确保 HTML 文件进入导入解析流程。

当前逻辑中，HTML 文件的 `fileCategory` 为 `'markup'`（非代码），会被跳过：

```javascript
// 当前逻辑（extract-import-map.mjs 第 535-537 行）
if (file.fileCategory !== 'code') {
  importMap[path] = [];
  continue;
}
```

**需要修改**：对 HTML 文件特殊处理，即使 `fileCategory` 为 `'markup'`，也执行导入解析：

```javascript
// 变更后
if (file.fileCategory !== 'code' && file.language !== 'html') {
  importMap[path] = [];
  continue;
}
```

#### 6.1.2 TS_JS_LANGS 无需变更

HTML 文件的导入解析不通过 `resolveTsJsImport()`，而是通过 HtmlPlugin 的 `resolveImports()` 直接提取 `<script src>` / `<link href>`。因此 `TS_JS_LANGS` 无需添加 `'html'`。

#### 6.1.3 HTML 特有的导入解析模式

1. **`<script src="./app.js">`** — 相对路径脚本引用，通过 HtmlPlugin.resolveImports() 解析
2. **`<link rel="stylesheet" href="./styles.css">`** — 相对路径样式表引用，通过 HtmlPlugin.resolveImports() 解析
3. **`<link rel="modulepreload" href="./module.js">`** — 模块预加载引用，通过 HtmlPlugin.resolveImports() 解析
4. **`<link rel="icon" href="./favicon.ico">`** — 关联资源（icon/prefetch/preconnect 等），不纳入 imports
5. **`<script src="https://cdn.example.com/lib.js">`** — 外部 CDN 引用，跳过（isExternalUrl 判定）
6. **`<link href="/absolute/path.css">`** — 绝对路径引用，解析为项目根目录相对路径
7. **内联 `<script>` / `<style>`** — 无外部引用，不产生 imports

#### 6.1.4 resolveImport 分发增强

```javascript
function resolveImport(imp, file, ctx) {
  const lang = file.language;
  const src = imp.source;
  if (TS_JS_LANGS.has(lang)) {
    const out = resolveTsJsImport(src, file, ctx);
    return out ? [out] : [];
  }
  // HTML 文件：script src / link href 直接作为相对路径解析
  if (lang === 'html') {
    const out = resolveHtmlImport(src, file, ctx);
    return out ? [out] : [];
  }
  return [];
}

/**
 * 解析 HTML 文件中的导入引用。
 * script src / link href 可能是相对路径或绝对路径。
 */
function resolveHtmlImport(rawImport, file, ctx) {
  if (!rawImport || typeof rawImport !== 'string') return null;
  const src = rawImport.trim();
  if (!src || isExternalUrl(src)) return null;

  const importerDir = dirOf(toPosix(file.path));

  if (src.startsWith('./') || src.startsWith('../')) {
    const base = resolveRelative(importerDir, src);
    return probeWithExtensions(base, ctx.fileSet) || (ctx.fileSet.has(base) ? base : null);
  }

  if (src.startsWith('/')) {
    // 绝对路径 → 项目根目录相对
    const base = src.slice(1);
    return probeWithExtensions(base, ctx.fileSet) || (ctx.fileSet.has(base) ? base : null);
  }

  // 裸路径（无前缀）→ 相对于当前目录
  const base = resolveRelative(importerDir, src);
  return probeWithExtensions(base, ctx.fileSet) || (ctx.fileSet.has(base) ? base : null);
}

function isExternalUrl(url) {
  return /^(https?:)?\/\//i.test(url) || /^(data:|blob:|mailto:|tel:|javascript:)/i.test(url);
}
```

## 10. 风险评估

| 风险 | 等级 | 影响 | 缓解措施 |
|---|---|---|---|
| `node-html-parser` 行号精度不足 | 中 | HTML 元素的 `lineRange` 可能不精确 | 使用 `range` 偏移量 + `Uint32Array` 行号映射表 O(1) 查找计算行号，误差在 ±1 行内 |
| `node.range` 返回 [-1,-1] | 高 | 当 `range` 选项未设置或解析失败时，`node.range` 返回 `[-1,-1]`（`enumerable: false`），直接使用会导致行号计算错误 | `computeLineRange` 中检测 `start === -1 || end === -1`，回退到 `fallbackLineRange` 从源文本搜索标签位置计算行号 |
| malformed HTML 解析不一致 | 低 | 不同浏览器的容错行为不同，node-html-parser 的容错可能与浏览器不一致 | 降级返回空分析，不影响其他文件解析 |
| HTML 文件 `fileCategory` 为 `markup` 被跳过 | 高 | extract-import-map.mjs 跳过非代码文件，HTML 的引用不会被解析 | 修改跳过逻辑，对 `language === 'html'` 特殊处理 |
| `<link>` 非 stylesheet rel 误入 imports | 高 | `<link rel="icon">` / `<link rel="prefetch">` 等不应作为代码依赖纳入 imports | `IMPORTABLE_LINK_RELS` 白名单过滤，仅 `stylesheet` / `modulepreload` 纳入 imports |
| `<script type="module">` 与 `<script>` 区分 | 低 | 两种 script 标签的引用语义不同 | 当前统一处理为 imports，后续可按 type 属性细分 |
| HTML 模板语法干扰解析 | 低 | 含模板语法（`{{ }}`、`<% %>`)的 HTML 可能解析失败 | 降级返回空分析，不影响其他文件 |
| `StructuralAnalysis` 新增字段向后兼容 | 低 | 旧版消费方可能不识别 `htmlElements` 字段 | 字段为 optional，旧版消费方忽略即可 |
