# P1.8: CSS-in-JS 支持详细设计规格

> 版本: 1.0
> 日期: 2026-06-05
> 状态: 已评审（修复 HIGH 问题后）
> 前置: P1.4（React JSX 语义提取器）已完成
> 变更: 扩展 ReactExtractor，新增 `identifyCssInJs()` 方法，识别 styled-components / Emotion / styled-jsx / JSS·MUI 四种主流 CSS-in-JS 模式

## 1. 目标与范围

在 ReactExtractor（P1.4）基础上增加 CSS-in-JS 模式识别，使项目对 React 代码库的分析能力从"组件/hooks/Context/HOC 语义感知"升级为"包含样式层语义的完整感知"，自动识别 styled-components、Emotion、styled-jsx、JSS/MUI 等主流 CSS-in-JS 方案的样式定义与消费关系。

**范围**：
- 扩展 `ReactExtractor`，新增 `identifyCssInJs()` 方法
- 新增 CSS-in-JS 相关 `tags` 标记（`css-in-js`、`styled-component`、`emotion`、`styled-jsx`、`jss`、`mui`），file-analyzer 可基于 tags 推断 `depends_on` 边
- 新增 LLM 分析提示词（`skills/understand/languages/css-in-js.md`）

**不在范围内**：
- CSS-in-JS 运行时行为分析（如动态主题切换）
- CSS-in-JS 样式内容提取（不解析模板字符串中的 CSS 规则）
- CSS-in-JS SSR/提取优化分析
- 其他 CSS-in-JS 方案（如 vanilla-extract、Panda CSS、Tailwind CSS — 后者不属于 CSS-in-JS 范畴）

## 2. CSS-in-JS 生态分析

### 2.1 四种主流方案对比

| 维度 | styled-components | Emotion | styled-jsx | JSS/MUI |
|---|---|---|---|---|
| **包名** | `styled-components` | `@emotion/styled` + `@emotion/react` + `@emotion/css` | 内置于 Next.js / `styled-jsx` | `@mui/styles`（已废弃）/ `jss` |
| **核心 API** | `styled.xxx`、`styled("xxx")`、`styled(Comp)` | `styled.xxx`、`css` 模板字面量、`css` prop | `<style jsx>`、`<style jsx global>` | `createStyles()`、`makeStyles()`、`withStyles()` |
| **样式定义方式** | 模板字面量 | 模板字面量 / 对象样式 | JSX 内联 `<style>` 标签 | JavaScript 对象 |
| **运行时** | 有（动态生成 CSS） | 可选（零运行时模式） | 有（编译时提取） | 有 |
| **TypeScript 支持** | 良好 | 良好 | 一般 | 一般 |
| **市场占比（2025）** | 高 | 高 | 中（Next.js 内置） | 低（MUI v5+ 推荐 sx/tss-react） |
| **识别难度** | 中 | 中 | 中 | 低 |

### 2.2 识别优先级

```
1. styled-components / Emotion — 共享 styled API，需通过导入源区分
2. styled-jsx — JSX 内联模式，特征明显
3. JSS/MUI — API 调用模式，特征明显
```

**关键区分点**：styled-components 和 Emotion 的 `styled.xxx` API 完全相同，需通过导入来源（`import styled from 'styled-components'` vs `import styled from '@emotion/styled'`）区分。

## 3. 识别规则设计

### 3.1 styled-components 模式

#### 3.1.1 正则预扫描

```typescript
const STYLED_COMPONENTS_PATTERNS = [
  // styled.xxx — 标签模板字面量
  /styled\.\w+[`(\s]/,
  // styled("xxx") / styled('xxx') — 字符串参数
  /styled\(['"][\w-]+['"]\)/,
  // styled(Comp) — 组件继承
  /styled\([A-Z]\w*\)/,
  // .attrs() 链式调用
  /\.attrs\s*\(/,
  // .extend — 旧版继承（v3）
  /\.extend[`(\s]/,
];
```

#### 3.1.2 AST 验证

```typescript
/**
 * AST 验证规则：
 *
 * 1. styled.xxx 模式
 *    call_expression
 *      → function: member_expression (styled.div / styled.button / ...)
 *      → arguments: template_string
 *
 * 2. styled("xxx") / styled('xxx') 模式
 *    call_expression
 *      → function: identifier "styled"
 *      → arguments: string (标签名)
 *    call_expression (外层)
 *      → arguments: template_string (样式内容)
 *
 * 3. styled(Comp) 模式
 *    call_expression
 *      → function: identifier "styled"
 *      → arguments: identifier (PascalCase 组件名)
 *    call_expression (外层)
 *      → arguments: template_string (样式内容)
 *
 * 4. .attrs() 链式调用
 *    call_expression
 *      → function: member_expression (.attrs)
 *    → 父级为 styled 调用链
 */
```

#### 3.1.3 导入源验证

```typescript
// 仅当从 'styled-components' 导入 styled 时标记为 styled-components
// 排除从 '@emotion/styled' 导入的同名 API
const isStyledComponentsImport = (result: StructuralAnalysis): boolean => {
  return result.imports.some(
    imp => imp.source === 'styled-components' && imp.specifiers.includes('styled')
  );
};
```

### 3.2 Emotion 模式

#### 3.2.1 正则预扫描

```typescript
const EMOTION_PATTERNS = [
  // css 模板字面量 — @emotion/css
  /(?:^|\s)css`/,
  // css({...}) 对象样式 — @emotion/css
  /css\s*\(\s*\{/,
  // css prop — JSX 属性（需 Babel 插件支持）
  /css=\{/,
];
```

#### 3.2.2 AST 验证

```typescript
/**
 * AST 验证规则：
 *
 * 1. css 模板字面量
 *    call_expression
 *      → function: identifier "css"
 *      → arguments: template_string
 *
 * 2. css({...}) 对象样式
 *    call_expression
 *      → function: identifier "css"
 *      → arguments: object
 *
 * 3. css prop（JSX 属性）
 *    jsx_attribute
 *      → name: identifier "css"
 *      → value: jsx_expression (对象或模板字面量)
 *
 * 4. styled.xxx（从 @emotion/styled 导入）
 *    — 与 styled-components 共享 AST 结构
 *    — 通过导入源区分
 */
```

#### 3.2.3 导入源验证

```typescript
const isEmotionImport = (result: StructuralAnalysis): boolean => {
  return result.imports.some(imp =>
    imp.source === '@emotion/styled' ||
    imp.source === '@emotion/react' ||
    imp.source === '@emotion/css'
  );
};

// 区分 Emotion styled vs styled-components styled
const isEmotionStyled = (result: StructuralAnalysis): boolean => {
  return result.imports.some(
    imp => imp.source === '@emotion/styled' && imp.specifiers.includes('styled')
  );
};
```

### 3.3 styled-jsx 模式

#### 3.3.1 导入源检测（主要方式）

TypeScriptExtractor 仅分析 script 内容，无法看到 JSX 模板中的 `<style jsx>` 标签。因此 styled-jsx 的检测必须基于导入语句，而非正则匹配 JSX 标签。

```typescript
const STYLED_JSX_IMPORT_SOURCES = [
  // 直接导入 styled-jsx
  'styled-jsx',
  // styled-jsx/css — 独立的 css 标签模板
  'styled-jsx/css',
];

// 检测方式：扫描 result.imports 中的 styled-jsx 相关导入
const isStyledJsxImport = (result: StructuralAnalysis): boolean => {
  return result.imports.some(imp =>
    imp.source === 'styled-jsx' || imp.source === 'styled-jsx/css'
  );
};
```

**说明**：

- `import css from 'styled-jsx/css'` 是 styled-jsx 提供的独立 CSS 标签模板 API，可在 script 中使用 `css\`...\``
- Next.js 项目中 styled-jsx 通常无需显式导入（全局可用），此时需依赖 AST 中 `<style jsx>` 标签检测（见 3.3.2）
- 正则预扫描不应匹配 `<style jsx>` 模式，因为 TypeScriptExtractor 无法看到 JSX 模板内容

#### 3.3.2 AST 验证

styled-jsx 有两种使用方式，对应不同的 AST 检测策略：

**方式 1：`styled-jsx/css` 导入的 `css` 标签模板（script 内可见）**

```typescript
/**
 * 当检测到 'styled-jsx/css' 导入时，在 AST 中查找 css 标签模板调用。
 *
 * import css from 'styled-jsx/css'
 * const styles = css`div { color: red }`
 *
 * AST 结构:
 *   variable_declarator
 *     → name: identifier
 *     → value: call_expression
 *       → function: identifier "css"
 *       → arguments: template_string
 */
```

**方式 2：`<style jsx>` JSX 标签（仅 tree-sitter JSX 解析可见）**

```typescript
/**
 * 当 tree-sitter 解析器包含 JSX 语法时，可检测 <style jsx> 标签。
 * 注意：TypeScriptExtractor 的 rootNode 可能不包含 JSX 元素节点，
 * 此检测作为辅助手段，不作为主要检测方式。
 *
 * AST 结构:
 *   jsx_element
 *     → opening_element: jsx_opening_element
 *       → name: identifier "style"
 *       → attributes: jsx_attribute
 *         → name: identifier "jsx"
 *     → children: jsx_expression (模板字面量)
 */
```

**检测优先级**：导入源检测（`styled-jsx` / `styled-jsx/css`）> AST 标签检测（`<style jsx>`）

#### 3.3.3 导入源验证

```typescript
// styled-jsx 的导入检测是主要识别方式
// 两种导入模式：
// 1. import styled-jsx（直接导入，通常用于类型或配置）
// 2. import css from 'styled-jsx/css'（css 标签模板 API）
const isStyledJsxImport = (result: StructuralAnalysis): boolean => {
  return result.imports.some(imp =>
    imp.source === 'styled-jsx' || imp.source === 'styled-jsx/css'
  );
};

// 即使无显式导入，AST 中存在 <style jsx> 也应识别
// （Next.js 项目中 styled-jsx 是全局可用的）
// 但此检测受限于 tree-sitter 是否解析 JSX 节点
```

### 3.4 JSS/MUI 模式

#### 3.4.1 正则预扫描

```typescript
const JSS_MUI_PATTERNS = [
  // createStyles() — JSS / @mui/styles
  /createStyles\s*\(/,
  // makeStyles() — @mui/styles
  /makeStyles\s*\(/,
  // withStyles() — @mui/styles
  /withStyles\s*\(/,

```

#### 3.4.2 AST 验证

```typescript
/**
 * AST 验证规则：
 *
 * 1. createStyles() 模式
 *    call_expression
 *      → function: identifier "createStyles"
 *      → arguments: object (样式规则对象)
 *
 * 2. makeStyles() 模式
 *    call_expression
 *      → function: identifier "makeStyles"
 *      → arguments:
 *        - arrow_function (theme => ({...}))

 *    call_expression
 *      → function: identifier "withStyles"
 *      → arguments:
 *        - object / arrow_function (样式定义)

 *      → arguments: identifier (组件名)
 */
```

```typescript
const isJssMuiImport = (result: StructuralAnalysis): boolean => {
  return result.imports.some(imp =>
    imp.source === '@mui/styles' ||
    imp.source === '@material-ui/styles' ||
    imp.source === 'jss' ||
    imp.source === 'react-jss'

};

// 区分 MUI vs 纯 JSS
const isMuiStyles = (result: StructuralAnalysis): boolean => {
```

## 4. ReactExtractor 扩展

### 4.1 新增 `identifyCssInJs()` 方法

**文件**: `packages/core/src/plugins/extractors/react-extractor.ts`

```typescript
/**
 * CSS-in-JS 模式识别常量
 */
const CSS_IN_JS_IMPORT_SOURCES = {
  'styled-components': ['styled-components'],
  'emotion': ['@emotion/styled', '@emotion/react', '@emotion/css'],
  'styled-jsx': ['styled-jsx', 'styled-jsx/css'],
  'jss': ['jss', 'react-jss'],
  'mui': ['@mui/styles', '@material-ui/styles'],
} as const;

/**
 * CSS-in-JS 样式定义信息
 */
interface CssInJsDefinition {
  name: string;              // 样式组件/变量名
  library: 'styled-components' | 'emotion' | 'styled-jsx' | 'jss' | 'mui';
  pattern: string;           // 识别的具体模式（如 'styled.xxx', 'css', 'makeStyles' 等）
  lineNumber: number;
  parentFunction?: string;   // 所属函数（如组件名）
}

export class ReactExtractor extends TypeScriptExtractor implements LanguageExtractor {
  // ... 现有方法 ...

  extractStructure(rootNode: TreeSitterNode): StructuralAnalysis {
    // 阶段 1: 调用父类提取基础结构
    const result = super.extractStructure(rootNode);

    // 阶段 2: 增量分析 React 模式
    this.identifyHooks(result, rootNode);
    this.identifyComponents(result, rootNode);
    this.identifyJsxComposition(result, rootNode);
    this.identifyContextRelations(result, rootNode);
    this.identifyHocPatterns(result, rootNode);
    // ★ P1.8 新增: CSS-in-JS 模式识别
    this.identifyCssInJs(result, rootNode);

    return result;
  }

  extractCallGraph(rootNode: TreeSitterNode): CallGraphEntry[] {
    // 阶段 1: 调用父类提取基础调用图
    const entries = super.extractCallGraph(rootNode);

    // 阶段 2: 增量分析 React 关系边
    this.addContainsEdges(entries, rootNode);
    this.addDependsOnEdges(entries, rootNode);
    // 注意：CSS-in-JS 的 depends_on 边不在此处生成。
    // ReactExtractor 仅产出 StructuralAnalysis（含 tags），不含边概念。
    // file-analyzer agent 根据 tags（如 'css-in-js'、'styled-component'）推断
    // 组件→样式定义的 depends_on 边。

    return entries;
  }

  /**
   * 识别 CSS-in-JS 模式。
   *
   * 识别流程:
   * 1. 扫描导入语句，确定项目中使用的 CSS-in-JS 库
   * 2. 遍历 AST，按库匹配对应的模式
   * 3. 标记函数/变量的 tags
   * 4. 标记导入的 importKind
   *
   * 交叉验证:
   * - styled.xxx 必须确认导入源（styled-components vs @emotion/styled）
   * - css 模板字面量必须确认从 @emotion/css 导入
   * - makeStyles/createStyles 必须确认从 @mui/styles 或 jss 导入
   */
  private identifyCssInJs(result: StructuralAnalysis, rootNode: TreeSitterNode): void {
    // 1. 扫描导入，构建 CSS-in-JS 库使用图
    const cssInJsLibraries = this.detectCssInJsLibraries(result);

    // 2. 如果未检测到任何 CSS-in-JS 库，跳过（避免无谓的 AST 遍历）
    if (cssInJsLibraries.size === 0) return;

    // 3. 遍历 AST 识别各库的模式
    const definitions = this.findCssInJsDefinitions(rootNode, cssInJsLibraries);

    // 4. 标记函数 tags
    for (const def of definitions) {
      for (const fn of result.functions) {
        if (fn.name === def.name) {
          fn.tags = [...(fn.tags ?? []), 'css-in-js', def.library];
          // 额外标记具体模式
          if (def.library === 'styled-components' || def.library === 'emotion') {
            if (def.pattern === 'styled.xxx' || def.pattern === 'styled(Comp)') {
              fn.tags = [...(fn.tags ?? []), 'styled-component'];
            }
          }
        }
      }
    }

    // 5. 标记 imports 的 importKind
    this.markCssInJsImports(result, cssInJsLibraries);
  }

  /**
   * 检测项目中使用的 CSS-in-JS 库。
   */
  private detectCssInJsLibraries(result: StructuralAnalysis): Map<string, string[]> {
    const libraries = new Map<string, string[]>();

    for (const imp of result.imports) {
      for (const [lib, sources] of Object.entries(CSS_IN_JS_IMPORT_SOURCES)) {
        if (sources.includes(imp.source as any)) {
          if (!libraries.has(lib)) {
            libraries.set(lib, []);
          }
          libraries.get(lib)!.push(...imp.specifiers);
        }
      }
    }

    // 特殊处理: styled-jsx 可能无显式导入（Next.js 内置）
    // 通过 AST 中 <style jsx> 标签检测
    // 此处不处理，在 findCssInJsDefinitions 中兜底

    return libraries;
  }

  /**
   * 查找 CSS-in-JS 样式定义。
   *
   * 消歧策略：styled-components 和 Emotion 共享 styled.xxx API，
   * 需通过导入源区分。detectCssInJsLibraries() 已预先扫描导入，
   * 此处根据检测结果分别标记。
   *
   * 当两个库同时导入时，findStyledPatterns 会为每个 styled 调用
   * 根据其绑定的导入源确定所属库（详见 findStyledPatterns 实现）。
   */
  private findCssInJsDefinitions(
    rootNode: TreeSitterNode,
    libraries: Map<string, string[]>,
  ): CssInJsDefinition[] {
    const definitions: CssInJsDefinition[] = [];

    // styled-components / Emotion styled 模式
    // 两个库可能同时存在，需分别处理
    if (libraries.has('styled-components') || libraries.has('emotion')) {
      definitions.push(...this.findStyledPatterns(rootNode, libraries));
    }

    // Emotion css 模式
    if (libraries.has('emotion')) {
      definitions.push(...this.findEmotionCssPatterns(rootNode));
    }

    // styled-jsx 模式（优先通过导入检测，AST 标签检测作为辅助）
    if (libraries.has('styled-jsx')) {
      definitions.push(...this.findStyledJsxPatterns(rootNode));
    }

    // JSS/MUI 模式
    if (libraries.has('jss') || libraries.has('mui')) {
      definitions.push(...this.findJssMuiPatterns(rootNode, libraries));
    }

    return definitions;
  }

  /**
   * 查找 styled.xxx / styled(Comp) 模式。
   *
   * 消歧逻辑：当 styled-components 和 @emotion/styled 同时导入时，
   * 需确定每个 styled 调用绑定的是哪个导入源。
   *
   * 策略：
   * 1. 如果仅导入一个库，所有 styled 调用归属于该库
   * 2. 如果两个库同时导入，通过变量绑定关系确定：
   *    - import styled from 'styled-components' → styled 绑定到 styled-components
   *    - import styled from '@emotion/styled' → styled 绑定到 emotion
   *    - 如果两个库都导入了 styled，使用最后导入的（ES module 覆盖语义）
   *    - 如果使用了别名（import scStyled from 'styled-components'），
   *      通过 specifier 与 AST 中标识符的匹配来确定
   */
  private findStyledPatterns(
    rootNode: TreeSitterNode,
    libraries: Map<string, string[]>,
  ): CssInJsDefinition[] {
    const definitions: CssInJsDefinition[] = [];

    // 确定默认库：当仅有一个 styled 库时使用
    const hasStyledComponents = libraries.has('styled-components');
    const hasEmotionStyled = libraries.has('emotion') &&
      libraries.get('emotion')?.some(s => s.includes('styled')) === true;
    const defaultLib: 'styled-components' | 'emotion' | null =
      hasStyledComponents && !hasEmotionStyled ? 'styled-components' :
      hasEmotionStyled && !hasStyledComponents ? 'emotion' : null;

    // 辅助函数：根据标识符名确定 styled 所属库
    const resolveStyledLibrary = (
      identifierText: string,
      fallback: 'styled-components' | 'emotion' | null,
    ): 'styled-components' | 'emotion' => {
      // 如果标识符不是 'styled'，可能是别名导入
      // 检查 libraries 中是否有对应的 specifier
      if (identifierText !== 'styled') {
        // 检查 styled-components 的 specifier
        const scSpecifiers = libraries.get('styled-components') ?? [];
        if (scSpecifiers.includes(identifierText)) return 'styled-components';
        // 检查 emotion 的 specifier
        const emSpecifiers = libraries.get('emotion') ?? [];
        if (emSpecifiers.includes(identifierText)) return 'emotion';
      }
      // 默认：styled 标识符使用 fallback（单库时确定，双库时默认 styled-components）
      return fallback ?? 'styled-components';
    };

    traverse(rootNode, (node) => {
      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (!func) return;

        // 模式 1: styled.div`...` / styled.button`...`
        if (func.type === 'member_expression') {
          const object = func.childForFieldName('object');
          const property = func.childForFieldName('property');
          if (object?.text === 'styled' && property) {
            const parent = node.parent;
            if (parent?.type === 'variable_declarator') {
              const nameNode = parent.childForFieldName('name');
              if (nameNode) {
                definitions.push({
                  name: nameNode.text,
                  library: resolveStyledLibrary(object.text, defaultLib),
                  pattern: 'styled.xxx',
                  lineNumber: node.startPosition.row + 1,
                });
              }
            }
          }
        }

        // 模式 2: styled("div")`...` / styled(Comp)`...`
        if (func.type === 'identifier' && func.text === 'styled') {
          const args = node.childForFieldName('arguments');
          if (args && args.children.length > 0) {
            const firstArg = args.children.find(c => c.type !== ',' && c.type !== '(' && c.type !== ')');
            if (firstArg) {
              const parent = node.parent;
              // styled(Comp) 可能被外层 call_expression 包裹（模板字面量参数）
              let variableParent = parent;
              if (parent?.type === 'call_expression') {
                variableParent = parent.parent;
              }
              if (variableParent?.type === 'variable_declarator') {
                const nameNode = variableParent.childForFieldName('name');
                if (nameNode) {
                  const isComponent = /^[A-Z]/.test(firstArg.text);
                  definitions.push({
                    name: nameNode.text,
                    library: resolveStyledLibrary(func.text, defaultLib),
                    pattern: isComponent ? 'styled(Comp)' : 'styled("xxx")',
                    lineNumber: node.startPosition.row + 1,
                  });
                }
              }
            }
          }
        }

        // 模式 3: .attrs() 链式调用
        if (func.type === 'member_expression') {
          const property = func.childForFieldName('property');
          if (property?.text === 'attrs') {
            const object = func.childForFieldName('object');
            if (object?.type === 'call_expression') {
              // 向上查找 variable_declarator 获取名称
              const parent = node.parent;
              let variableParent = parent;
              if (parent?.type === 'call_expression') {
                variableParent = parent.parent;
              }
              if (variableParent?.type === 'variable_declarator') {
                const nameNode = variableParent.childForFieldName('name');
                if (nameNode) {
                  definitions.push({
                    name: nameNode.text,
                    library: resolveStyledLibrary(object.text, defaultLib),
                    pattern: 'styled.attrs',
                    lineNumber: node.startPosition.row + 1,
                  });
                }
              }
            }
          }
        }
      }
    });

    return definitions;
  }

  /**
   * 查找 Emotion css 模板字面量模式。
   */
  private findEmotionCssPatterns(rootNode: TreeSitterNode): CssInJsDefinition[] {
    const definitions: CssInJsDefinition[] = [];

    traverse(rootNode, (node) => {
      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (func?.type === 'identifier' && func.text === 'css') {
          const parent = node.parent;
          if (parent?.type === 'variable_declarator') {
            const nameNode = parent.childForFieldName('name');
            if (nameNode) {
              definitions.push({
                name: nameNode.text,
                library: 'emotion',
                pattern: 'css',
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
          // css`...` 作为对象属性值或参数使用（不赋值给变量）
          // 仍然标记为 emotion 模式，但无变量名
        }
      }

      // css prop 模式: <div css={{...}}>
      if (node.type === 'jsx_attribute') {
        const nameNode = node.childForFieldName('name');
        if (nameNode?.text === 'css') {
          definitions.push({
            name: '<css-prop>',
            library: 'emotion',
            pattern: 'css-prop',
            lineNumber: node.startPosition.row + 1,
          });
        }
      }
    });

    return definitions;
  }

  /**
   * 查找 styled-jsx 模式。
   *
   * 检测策略：
   * 1. 优先检测 styled-jsx/css 导入的 css 标签模板（script 内可见）
   * 2. 辅助检测 <style jsx> JSX 标签（受限于 tree-sitter 是否解析 JSX 节点）
   */
  private findStyledJsxPatterns(rootNode: TreeSitterNode): CssInJsDefinition[] {
    const definitions: CssInJsDefinition[] = [];

    traverse(rootNode, (node) => {
      // 方式 1: styled-jsx/css 的 css 标签模板
      // import css from 'styled-jsx/css'
      // const styles = css`div { color: red }`
      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (func?.type === 'identifier' && func.text === 'css') {
          const parent = node.parent;
          if (parent?.type === 'variable_declarator') {
            const nameNode = parent.childForFieldName('name');
            if (nameNode) {
              definitions.push({
                name: nameNode.text,
                library: 'styled-jsx',
                pattern: 'css-tag',
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
        }
      }

      // 方式 2: <style jsx> JSX 标签（辅助检测）
      // 注意：TypeScriptExtractor 的 rootNode 可能不包含 JSX 元素节点
      if (node.type === 'jsx_opening_element') {
        const nameNode = node.childForFieldName('name');
        if (nameNode?.text === 'style') {
          // 检查是否有 jsx 属性
          const attributes = findChildren(node, 'jsx_attribute');
          const hasJsxAttr = attributes.some(attr => {
            const attrName = attr.childForFieldName('name');
            return attrName?.text === 'jsx';
          });
          const hasGlobalAttr = attributes.some(attr => {
            const attrName = attr.childForFieldName('name');
            return attrName?.text === 'global';
          });

          if (hasJsxAttr) {
            definitions.push({
              name: '<style jsx>',
              library: 'styled-jsx',
              pattern: hasGlobalAttr ? 'style-jsx-global' : 'style-jsx',
              lineNumber: node.startPosition.row + 1,
            });
          }
        }
      }
    });

    return definitions;
  }

  /**
   * 查找 JSS/MUI 模式。
   */
  private findJssMuiPatterns(
    rootNode: TreeSitterNode,
    libraries: Map<string, string[]>,
  ): CssInJsDefinition[] {
    const definitions: CssInJsDefinition[] = [];
    const isMui = libraries.has('mui');

    traverse(rootNode, (node) => {
      if (node.type === 'call_expression') {
        const func = node.childForFieldName('function');
        if (!func) return;

        const funcText = func.text;

        // makeStyles() 模式
        if (funcText === 'makeStyles') {
          const parent = node.parent;
          if (parent?.type === 'variable_declarator') {
            const nameNode = parent.childForFieldName('name');
            if (nameNode) {
              definitions.push({
                name: nameNode.text,
                library: isMui ? 'mui' : 'jss',
                pattern: 'makeStyles',
                lineNumber: node.startPosition.row + 1,
              });
            }
          }
        }

        // createStyles() 模式
        if (funcText === 'createStyles') {
          definitions.push({
            name: 'createStyles',
            library: isMui ? 'mui' : 'jss',
            pattern: 'createStyles',
            lineNumber: node.startPosition.row + 1,
          });
        }

        // withStyles() 模式（HOC 风格）
        if (funcText === 'withStyles') {
          const args = node.childForFieldName('arguments');
          if (args) {
            // withStyles 返回 HOC，可能被外层调用包裹组件
            definitions.push({
              name: 'withStyles',
              library: isMui ? 'mui' : 'jss',
              pattern: 'withStyles',
              lineNumber: node.startPosition.row + 1,
            });
          }
        }
      }
    });

    return definitions;
  }

  /**
   * 标记 CSS-in-JS 相关导入的 importKind。
   */
  private markCssInJsImports(result: StructuralAnalysis, libraries: Map<string, string[]>): void {
    for (const imp of result.imports) {
      for (const [lib, sources] of Object.entries(CSS_IN_JS_IMPORT_SOURCES)) {
        if (sources.includes(imp.source as any)) {
          imp.importKind = 'css-in-js';
          break;
        }
      }
    }
  }

  /**
   * CSS-in-JS 依赖边推断策略。
   *
   * ReactExtractor 仅产出 StructuralAnalysis（含 tags），不直接生成边。
   * depends_on 边由 file-analyzer agent 根据 tags 推断：
   *
   * 推断规则：
   * - 函数 tags 含 'css-in-js' + 'styled-component' → 该函数为样式组件，
   *   引用该组件的 JSX 元素与该函数形成 depends_on 边
   * - 函数 tags 含 'css-in-js' + 'jss'/'mui' → 该函数为样式 hook（如 useStyles），
   *   调用该 hook 的组件与该函数形成 depends_on 边
   * - imports 的 importKind 为 'css-in-js' → 组件与 CSS-in-JS 库形成 depends_on 边
   *
   * 此方法已移除，保留此注释作为设计决策记录。
   * 实际边生成逻辑在 file-analyzer agent 中实现。
   */
}
```

### 4.3 与 `identifyComponents()` 的协作

CSS-in-JS 样式组件（如 `const Button = styled.button\`...\``）同时满足两个条件：
1. PascalCase 命名（`Button`）
2. 可能返回类 JSX 元素（styled-components 生成的组件在运行时渲染为 DOM 元素）

**协作策略**：

```typescript
// identifyCssInJs() 在 identifyComponents() 之后执行
// 如果函数已被标记为 css-in-js，则不再标记为 component
// 理由: styled-component 的语义是"样式定义"，而非"UI 组件"
// 但保留 component 标签，因为 styled-component 在 JSX 中作为组件使用

// 标记优先级:
// 1. 如果已被标记为 hook → 不标记（hook 优先级最高）
// 2. 如果已被标记为 component → 追加 css-in-js 标签
// 3. 如果仅匹配 css-in-js 模式 → 标记 css-in-js + styled-component
```

**具体实现**：

```typescript
// 在 identifyCssInJs() 中
for (const fn of result.functions) {
  if (fn.name === def.name) {
    const existingTags = fn.tags ?? [];
    // 如果已被标记为 component，追加 css-in-js 标签
    if (existingTags.includes('component')) {
      fn.tags = [...existingTags, 'css-in-js', def.library];
    } else {
      // 否则，标记为 css-in-js + styled-component
      fn.tags = [...existingTags, 'css-in-js', def.library, 'styled-component'];
    }
  }
}
```

## 5. Layer 3: LLM 指南

### 5.1 文件: `skills/understand/languages/css-in-js.md`

```markdown
# CSS-in-JS Language Prompt Snippet

## Key Concepts

- **CSS-in-JS**: A styling technique where CSS is written in JavaScript, enabling dynamic styling, scoping, and co-location of styles with components
- **styled-components**: Tagged template literal API (`styled.xxx`) that generates React components with scoped styles; supports theming via `ThemeProvider`, props-based dynamic styles, and component inheritance via `styled(Comp)`
- **Emotion**: Flexible CSS-in-JS library with multiple APIs — `styled` (similar to styled-components), `css` template literal / object styles, and the `css` JSX prop; supports framework-agnostic usage via `@emotion/css`
- **styled-jsx**: Built-in CSS-in-JS for Next.js; uses `<style jsx>` tags inside JSX for scoped styles and `<style jsx global>` for global styles; full CSS syntax support
- **JSS**: JavaScript-to-CSS compiler using object notation; `createStyles()` for type-safe style definitions; powers MUI's legacy styling API
- **MUI Styles**: Material-UI's JSS-based styling API (`makeStyles`, `withStyles`, `createStyles`); deprecated in MUI v5+ in favor of the `sx` prop and `tss-react`
- **Theming**: CSS-in-JS libraries provide theme context (`ThemeProvider`) for consistent design tokens across components
- **Dynamic Styles**: Props-driven style computation (e.g., `color: ${props => props.primary ? 'blue' : 'gray'}`)
- **Style Composition**: Combining multiple style definitions via composition patterns (e.g., `styled(StyledComp)` for inheritance, Emotion's composition with `css`)

## Notable File Patterns

- `*.styles.ts` — Style definition files (common in Emotion/JSS projects)
- `*.styled.ts` — styled-components definition files
- `styles/` — Directory containing style definitions
- `theme.ts` / `theme.tsx` — Theme configuration files
- `useStyles.ts` — Custom hook files for JSS/MUI makeStyles
- `*.emotion.ts` — Emotion-specific style files

## Edge Patterns

- `styled.div` creates a `styled-component` that is both a React component and a style definition — `contains` edge to rendered elements, `depends_on` edge to theme context (inferred by file-analyzer from tags)
- `styled(ExistingComponent)` creates a `styled-component` with an `inherits` edge to `ExistingComponent` (inferred by file-analyzer from tags)
- `css` template literal from Emotion creates a style definition — `depends_on` edge to the consuming component (inferred by file-analyzer from tags)
- `<style jsx>` inside a component creates a `depends_on` edge from the component to its scoped styles (inferred by file-analyzer from tags)
- `makeStyles()` creates a style hook — `depends_on` edge from the consuming component to the style definition, and to the theme if used (inferred by file-analyzer from tags)
- `withStyles()` is both a HOC and a style definition — `depends_on` edge to the style definition and `hoc-wrapped` tag on the wrapped component (inferred by file-analyzer from tags)
- `ThemeProvider` creates a `context-definition` for the theme — `depends_on` edges from styled-components that consume theme values (inferred by file-analyzer from tags)
- `.attrs()` on a styled-component creates a `related` edge to the attribute configuration (inferred by file-analyzer from tags)
- CSS-in-JS imports (`styled-components`, `@emotion/styled`, etc.) are marked with `importKind: 'css-in-js'` — file-analyzer uses this to infer library dependency edges

## Summary Style

> "styled-component defining a Button with primary/secondary variants, consumes theme colors via props-based dynamic styles."
> "Emotion css prop usage in a form component, applying dynamic validation styles based on field state."
> "styled-jsx scoped styles for a Next.js page component, using global theme variables via :global() selector."
> "MUI makeStyles hook defining table row styles with theme-based spacing and color tokens."
```

## 9. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|
| styled-components 与 Emotion 混淆 | 中 | 标签标记为错误库 | 通过导入源严格区分；同一文件中同时导入两个库时，按导入源分别标记 |
| styled-jsx 无显式导入时漏识别 | 中 | Next.js 项目中 styled-jsx 未被识别 | 优先通过 `styled-jsx/css` 导入检测；AST 中 `<style jsx>` 标签作为辅助检测（受限于 tree-sitter 是否解析 JSX 节点）；无导入且 AST 不可见时无法识别，属已知限制 |
| MUI v5+ 新 API 未覆盖 | 中 | `sx` prop / `tss-react` 模式未识别 | P1.8 仅覆盖 JSS/MUI 经典 API（`makeStyles`/`withStyles`），`sx` prop 和 `tss-react` 留待后续迭代 |
| CSS-in-JS 动态样式无法静态分析 | 高 | `${props => ...}` 中的动态样式无法提取 | P1.8 仅识别模式并标记标签，不解析模板字符串内容；动态样式分析需要运行时支持，超出范围 |
