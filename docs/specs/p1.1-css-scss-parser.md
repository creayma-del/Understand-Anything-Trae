# P1.1: CSS/SCSS Parser 详细设计规格

> 版本: 1.0
> 日期: 2026-06-05
> 状态: 已评审（修复 HIGH 问题后）
> 变更: 初始版本 — 使用 PostCSS + postcss-scss 实现 CSS/SCSS 文件的确定性结构化分析，提取规则/变量/mixin/function 及导入关系

## 1. 目标与范围

为项目新增 CSS/SCSS 文件的确定性结构化分析能力，包括：

- **类型扩展**：在 `StructuralAnalysis` 中新增 `cssRules` 字段，承载 CSS 规则级结构信息
- **CssPlugin 实现**：基于 PostCSS + postcss-scss 的独立 AnalyzerPlugin，提取规则/变量/mixin/function
- **导入解析增强**：支持 `@import`/`@use`/`@forward` 解析 + SCSS partial 路径探测
- **降级处理**：PostCSS 解析失败时返回空分析，不中断整体流程
- **注册集成**：在 parsers/index.ts 中注册 CssPlugin

**范围边界**：

- ✅ `.css`、`.scss` 文件的完整结构化分析
- ✅ SCSS 特有语法（`$variable`、`@mixin`、`@function`、`@use`、`@forward`、`%placeholder`）
- ✅ 导入解析（`@import`/`@use`/`@forward` + partial `_` 前缀探测）
- ❌ `.less` 文件的深度分析（仅作为 CSS 基础规则提取，Less 特有语法暂不支持）
- ❌ `.sass` 缩进语法的深度分析（仅作为 CSS 基础规则提取）
- ❌ PostCSS 插件生态的运行时行为分析

## 2. CSS/SCSS 文件结构分析

CSS/SCSS 文件的核心结构单元是**规则块**（rule），包括：

```scss
// 变量定义
$primary-color: #3366cc;
$font-stack: ('Helvetica', 'Arial', sans-serif);

// Mixin 定义
@mixin respond-to($breakpoint) {
  @media (min-width: $breakpoint) {
    @content;
  }
}

// Function 定义
@function calculate-rem($px) {
  @return ($px / 16px) * 1rem;
}

// 规则块
.button {
  color: $primary-color;
  font-size: calculate-rem(14px);

  &:hover {
    color: darken($primary-color, 10%);
  }

  @include respond-to(768px) {
    padding: 12px 24px;
  }
}

// At-rule
@layer components {
  .card { padding: 16px; }
}

// 导入
@use 'base' as *;
@forward 'themes';
@import 'reset.css';
```

### 2.1 提取策略：PostCSS + postcss-scss

**方案选择说明**：CSS/SCSS 不使用 tree-sitter（无成熟的 CSS/SCSS WASM grammar 可用），而是使用 PostCSS 作为 AST 解析器。PostCSS 是业界标准的 CSS 解析库，postcss-scss 插件使其支持 SCSS 语法。

```
阶段 1: PostCSS 解析
  → postcss().process(content, { syntax: postcss-scss })
  → 生成 PostCSS AST（Root → Rule / AtRule / Declaration）

阶段 2: AST 遍历提取
  → Rule 节点 → 提取 selector + declarations（type: 'rule'）
  → AtRule 节点 → 按 name 分类：
      - @mixin → type: 'mixin'，提取参数
      - @function → type: 'function'，提取参数
      - @media / @layer / @keyframes / @supports → type: 'at-rule'
      - @import / @use / @forward → 提取到 imports
  → Declaration 节点 → 检测 $variable 定义（prop 以 $ 开头）

阶段 3: 行号映射
  → PostCSS 节点自带 source.start.line / source.end.line
  → 注意：部分节点（如 AtRule）可能缺少 source 属性，需运行时守卫
  → 直接映射为 lineRange（跳过无 source 的节点）
```

### 2.2 为什么不使用 tree-sitter

- tree-sitter-css / tree-sitter-scss 的 WASM grammar 不可用或质量不足
- PostCSS 是 CSS 生态的事实标准解析器，对 SCSS 语法支持完善
- postcss-scss 是 PostCSS 官方推荐的 SCSS 语法插件，维护活跃
- 与 VueSfcPlugin 模式一致：不依赖 tree-sitter，使用领域专用解析器

## 4. Layer 2: CssPlugin 实现

### 4.1 架构决策：独立 AnalyzerPlugin

与 VueSfcPlugin 模式一致，创建独立的 `CssPlugin`，原因：

1. CSS 语言配置不包含 `treeSitter` 字段，TreeSitterPlugin 不会处理 CSS 文件
2. CSS/SCSS 需要专用的 AST 解析器（PostCSS），而非 tree-sitter
3. 独立插件遵循项目已有的插件模式（MarkdownParser、YAMLConfigParser、VueSfcPlugin 等）

### 4.2 新增类型：CssRuleInfo

**文件**: `packages/core/src/types.ts`

```typescript
/**
 * CSS/SCSS 规则信息，用于 StructuralAnalysis.cssRules 字段。
 * 涵盖普通规则、at-rule、mixin、function 和变量定义。
 */
export interface CssRuleInfo {
  /** CSS 选择器或 at-rule 标识符（如 ".button"、"@media (min-width: 768px)"） */
  selector: string;
  /** 规则在文件中的行范围 [起始行, 结束行] */
  lineRange: [number, number];
  /** 规则内包含的声明属性名列表（如 ["color", "font-size"]）。
   *  对于 type: 'variable'，此字段为空数组 []，变量名存储在 selector 中，
   *  以保持与其他类型（存储属性名）的语义一致性。 */
  declarations: string[];
  /** 规则类型 */
  type: 'rule' | 'at-rule' | 'mixin' | 'function' | 'variable';
}
```

### 4.4 analyzeFile() 完整实现

```typescript
analyzeFile(filePath: string, content: string): StructuralAnalysis {
  const cssRules: CssRuleInfo[] = [];
  const imports: Array<{ source: string; specifiers: string[]; lineNumber: number }> = [];
  const exports: Array<{ name: string; lineNumber: number; isDefault?: boolean }> = [];

  try {
    const result = postcss().process(content, {
      syntax: scss,
      from: filePath,
    });

    // 遍历 AST
    result.root.walk((node) => {
      // 运行时守卫：PostCSS AtRule 等节点可能缺少 source 属性
      // 跳过无 source 信息的节点（行号不可用），不中断遍历
      if (!node.source?.start || !node.source?.end) return;

      switch (node.type) {
        case "rule": {
          // 普通 CSS 规则
          cssRules.push({
            selector: node.selector,
            lineRange: [node.source.start.line, node.source.end.line],
            declarations: extractDeclarations(node),
            type: "rule",
          });
          break;
        }
        case "atrule": {
          const atName = node.name; // "media", "mixin", "function", "use", "forward", "import", etc.

          if (atName === "mixin") {
            // @mixin 定义
            cssRules.push({
              selector: `@mixin ${extractMixinName(node)}`,
              lineRange: [node.source.start.line, node.source.end.line],
              declarations: extractDeclarations(node),
              type: "mixin",
            });
            // mixin 名称作为导出
            exports.push({
              name: extractMixinName(node),
              lineNumber: node.source.start.line,
            });
          } else if (atName === "function") {
            // @function 定义
            cssRules.push({
              selector: `@function ${extractFunctionName(node)}`,
              lineRange: [node.source.start.line, node.source.end.line],
              declarations: extractDeclarations(node),
              type: "function",
            });
            // function 名称作为导出
            exports.push({
              name: extractFunctionName(node),
              lineNumber: node.source.start.line,
            });
          } else if (atName === "import") {
            // @import 语句
            const importPath = stripQuotes(node.params);
            imports.push({
              source: importPath,
              specifiers: [],
              lineNumber: node.source.start.line,
            });
          } else if (atName === "use") {
            // @use 语句
            const { path: usePath, namespace } = parseUseDirective(node.params);
            imports.push({
              source: usePath,
              specifiers: namespace ? [namespace] : [],
              lineNumber: node.source.start.line,
            });
          } else if (atName === "forward") {
            // @forward 语句
            const forwardPath = stripQuotes(node.params.split(/\s+as\s+|\s+hide\s+|\s+show\s+/)[0]);
            imports.push({
              source: forwardPath,
              specifiers: [],
              lineNumber: node.source.start.line,
            });
          } else {
            // 其他 at-rule（@media, @keyframes, @layer, @supports 等）
            cssRules.push({
              selector: `@${atName} ${node.params}`,
              lineRange: [node.source.start.line, node.source.end.line],
              declarations: extractDeclarations(node),
              type: "at-rule",
            });
          }
          break;
        }
        case "decl": {
          // SCSS 变量定义（$variable: value）
          if (node.prop.startsWith("$")) {
            cssRules.push({
              selector: node.prop,
              lineRange: [node.source.start.line, node.source.end.line],
              declarations: [],
              type: "variable",
            });
            // 变量名作为导出
            exports.push({
              name: node.prop,
              lineNumber: node.source.start.line,
            });
          }
          break;
        }
      }
    });
  } catch (err) {
    // [降级守卫] PostCSS 解析失败时返回空分析
    // 不中断整体流程，仅记录警告
    return {
      functions: [],
      classes: [],
      imports: [],
      exports: [],
      cssRules: [],
    };
  }

  return {
    functions: [],
    classes: [],
    imports,
    exports,
    cssRules,
  };
}
```

### 4.5 辅助函数

```typescript
/**
 * 从规则节点中提取声明属性名列表。
 */
function extractDeclarations(node: postcss.Container): string[] {
  const declarations: string[] = [];
  if ("nodes" in node && node.nodes) {
    for (const child of node.nodes) {
      if (child.type === "decl") {
        declarations.push(child.prop);
      }
    }
  }
  return declarations;
}

/**
 * 从 @mixin 节点中提取 mixin 名称。
 * @mixin name($param1, $param2) { ... } → "name"
 */
function extractMixinName(node: postcss.AtRule): string {
  const match = node.params.match(/^([a-zA-Z_-][a-zA-Z0-9_-]*)/);
  return match ? match[1] : node.params;
}

/**
 * 从 @function 节点中提取 function 名称。
 * @function name($param) { ... } → "name"
 */
function extractFunctionName(node: postcss.AtRule): string {
  const match = node.params.match(/^([a-zA-Z_-][a-zA-Z0-9_-]*)/);
  return match ? match[1] : node.params;
}

/**
 * 去除字符串两端的引号。
 */
function stripQuotes(str: string): string {
  return str.replace(/^['"]|['"]$/g, "").trim();
}

/**
 * 解析 @use 指令的参数。
 * @use 'path' as name → { path: 'path', namespace: 'name' }
 * @use 'path' → { path: 'path', namespace: null }
 */
function parseUseDirective(params: string): { path: string; namespace: string | null } {
  const parts = params.split(/\s+as\s+/);
  const path = stripQuotes(parts[0]);
  const namespace = parts.length > 1 ? parts[1].trim() : null;
  return { path, namespace };
}
```

### 4.6 resolveImports() 实现

```typescript
resolveImports(filePath: string, content: string): ImportResolution[] {
  const analysis = this.analyzeFile(filePath, content);

  return analysis.imports.map(imp => {
    let resolvedPath = imp.source;
    let isVerified = false;

    // 相对路径解析
    if (resolvedPath.startsWith('./') || resolvedPath.startsWith('../')) {
      // SCSS partial 路径探测：按 Dart Sass 规范顺序生成候选路径
      const candidates = resolveScssPartialPaths(resolvedPath);
      // 验证候选路径是否存在（需配合文件系统访问）
      // 注意：CssPlugin 本身不直接访问文件系统，此处标记为未验证
      // 实际验证由 extract-import-map.mjs 中的 resolveCssImport() 完成
      resolvedPath = candidates[0] ?? imp.source;
      isVerified = false; // 候选路径未经文件系统验证
    } else {
      // 非相对路径（如包名）无法在此验证
      isVerified = false;
    }

    return {
      source: imp.source,
      resolvedPath,
      specifiers: imp.specifiers,
      isVerified,
    };
  });
}

/**
 * SCSS partial 路径探测（遵循 Dart Sass 规范）。
 *
 * SCSS 约定 partial 文件以 _ 前缀命名（如 _variables.scss），
 * 导入时可省略 _ 前缀和扩展名（如 @use 'variables'）。
 * 此函数按 Dart Sass 规范顺序生成所有可能的路径候选。
 *
 * @use "components/button" probing order (per Dart Sass):
 *   1. components/button.scss
 *   2. components/button.sass
 *   3. components/_button.scss  (partial)
 *   4. components/_button.sass  (partial)
 *   5. components/button/index.scss
 *   6. components/button/_index.scss
 *   7. components/button.css
 */
function resolveScssPartialPaths(rawPath: string): string[] {
  const candidates: string[] = [];
  const dir = rawPath.includes('/')
    ? rawPath.substring(0, rawPath.lastIndexOf('/') + 1)
    : '';
  const base = rawPath.includes('/')
    ? rawPath.substring(rawPath.lastIndexOf('/') + 1)
    : rawPath;

  // 如果已有扩展名，直接使用
  if (/\.(scss|sass|css)$/.test(base)) {
    candidates.push(rawPath);
    // 也尝试 _ 前缀版本
    if (!base.startsWith('_')) {
      candidates.push(dir + '_' + base);
    }
    return candidates;
  }

  // 无扩展名：按 Dart Sass 规范顺序生成候选
  // 1. 非partial .scss / .sass
  candidates.push(dir + base + '.scss');
  candidates.push(dir + base + '.sass');

  // 2. partial（_ 前缀）.scss / .sass
  if (!base.startsWith('_')) {
    candidates.push(dir + '_' + base + '.scss');
    candidates.push(dir + '_' + base + '.sass');
  }

  // 3. index 文件（仅 .scss）
  candidates.push(rawPath + '/index.scss');
  candidates.push(rawPath + '/_index.scss');

  // 4. .css 最后（Dart Sass 将 .css 视为最低优先级）
  candidates.push(dir + base + '.css');

  return candidates;
}
```

### 4.7 降级处理

CssPlugin 的降级策略：

| 场景 | 处理方式 |
|---|---|
| PostCSS 解析失败（语法错误） | 返回空 `StructuralAnalysis`（`cssRules: []`），不抛出异常 |
| 文件内容为空 | 正常解析，返回空 `cssRules` |
| `.less` 文件 | 使用 postcss-scss 尽力解析，Less 特有语法可能丢失 |
| `.sass` 缩进语法 | 使用 postcss-scss 尽力解析，缩进语法可能解析失败，降级为空分析 |
| 节点缺少 `source` 信息 | 跳过该节点（`!node.source?.start || !node.source?.end` 守卫），不中断遍历 |

#### 6.1.2 CSS 导入解析器

```javascript
/**
 * SCSS partial 路径探测扩展名列表（遵循 Dart Sass 规范顺序）。
 * 与 TS_EXT_PROBES 类似，但针对 CSS/SCSS 文件的命名约定。
 *
 * Dart Sass 对 @use "components/button" 的探测顺序：
 *   1. components/button.scss
 *   2. components/button.sass
 *   3. components/_button.scss  (partial)
 *   4. components/_button.sass  (partial)
 *   5. components/button/index.scss
 *   6. components/button/_index.scss
 *   7. components/button.css
 */
const CSS_EXT_PROBES = [
  '.scss', '.sass',
  '_index.scss',
  '.css',
];

/**
 * SCSS partial 前缀探测。
 * @use 'base' 可能对应 _base.scss
 */
const CSS_PARTIAL_PROBES = [
  // 先尝试带 _ 前缀的 partial 文件
  (base) => '_' + base,
];

/**
 * 解析 CSS/SCSS 导入路径。
 * 处理 SCSS partial 约定（_ 前缀 + 省略扩展名）。
 */
function resolveCssImport(rawImport, file, ctx) {
  if (!rawImport || typeof rawImport !== 'string') return null;
  const src = rawImport.trim();
  if (!src) return null;

  // 仅处理相对路径
  if (!src.startsWith('./') && !src.startsWith('../')) return null;

  const importerDir = dirOf(toPosix(file.path));
  const base = resolveRelative(importerDir, src);

  // 如果已有扩展名且存在于文件集中，直接返回
  if (ctx.fileSet.has(base)) return base;

  // 生成 SCSS partial 候选路径
  const dir = base.includes('/') ? base.substring(0, base.lastIndexOf('/') + 1) : '';
  const name = base.includes('/') ? base.substring(base.lastIndexOf('/') + 1) : base;

  // 如果已有扩展名
  if (/\.(scss|sass|css)$/.test(name)) {
    // 尝试 _ 前缀版本
    if (!name.startsWith('_')) {
      const candidate = dir + '_' + name;
      if (ctx.fileSet.has(candidate)) return candidate;
    }
    return null;
  }

  // 无扩展名：按 Dart Sass 规范顺序探测所有候选
  // 1. 非partial .scss / .sass
  for (const ext of ['.scss', '.sass']) {
    const candidate = dir + name + ext;
    if (ctx.fileSet.has(candidate)) return candidate;
  }

  // 2. partial（_ 前缀）.scss / .sass
  if (!name.startsWith('_')) {
    for (const ext of ['.scss', '.sass']) {
      const candidate = dir + '_' + name + ext;
      if (ctx.fileSet.has(candidate)) return candidate;
    }
  }

  // 3. index 文件（仅 .scss）
  if (ctx.fileSet.has(base + '/index.scss')) return base + '/index.scss';
  if (ctx.fileSet.has(base + '/_index.scss')) return base + '/_index.scss';

  // 4. .css 最后（Dart Sass 将 .css 视为最低优先级）
  const cssCandidate = dir + name + '.css';
  if (ctx.fileSet.has(cssCandidate)) return cssCandidate;

  return null;
}
```

#### 6.1.3 分发器更新

```javascript
function resolveImport(imp, file, ctx) {
  const lang = file.language;
  const src = imp.source;
  if (TS_JS_LANGS.has(lang)) {
    const out = resolveTsJsImport(src, file, ctx);
    return out ? [out] : [];
  }
  // 新增：CSS/SCSS 导入解析
  if (CSS_LANGS.has(lang)) {
    const out = resolveCssImport(src, file, ctx);
    return out ? [out] : [];
  }
  return [];
}
```

#### 6.1.4 CSS 文件分类变更

当前 `scan-project.mjs` 将 `.css`/`.scss`/`.sass`/`.less` 文件分类为 `'markup'`（非代码）。CssPlugin 注册后，这些文件需要被 `extract-import-map.mjs` 处理。

**方案**：在 `extract-import-map.mjs` 中，对 CSS 语言文件跳过 `fileCategory !== 'code'` 的过滤，或者在 `scan-project.mjs` 中将 CSS 文件分类改为 `'code'`。

**推荐方案**：修改 `scan-project.mjs` 中的 `EXT_TO_CATEGORY` 映射，将 CSS 相关扩展名分类为 `'code'`：

```javascript
// 变更前
'.css': 'markup',
'.scss': 'markup',
'.sass': 'markup',
'.less': 'markup',

// 变更后
'.css': 'code',
'.scss': 'code',
'.sass': 'code',
'.less': 'code',
```

**理由**：CSS/SCSS 文件现在具备结构化分析能力（CssPlugin），应被视为代码文件以参与导入解析和结构图构建。

## 10. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|---|---|---|---|
| postcss-scss 对 `.less` 文件解析不准确 | Less 特有语法（如变量 `@var`、mixin `.mixin()`）可能被误解析 | 中 | 在文档中明确 `.less` 仅支持基础 CSS 规则提取；未来可引入 postcss-less 插件 |
| postcss-scss 对 `.sass` 缩进语法解析失败 | 缩进语法与 SCSS 差异大，可能解析错误 | 高 | 降级为空分析；未来可引入专门的 sass 解析器 |
| PostCSS 解析性能 | 大型 CSS 文件（>5000 行）解析可能较慢 | 低 | PostCSS 解析速度通常足够（~10ms/1000行）；如需优化可考虑缓存 |
| `CssRuleInfo` 类型扩展影响现有代码 | 新增 `cssRules` 字段为可选，不影响现有接口 | 低 | 可选字段确保向后兼容 |
| SCSS partial 路径探测不完整 | 某些项目使用自定义导入路径（如 `node_modules` 中的包） | 中 | `resolveImports()` 仅处理相对路径；包路径解析留给后续迭代 |
| `extract-import-map.mjs` 中 CSS 文件分类变更 | 将 CSS 从 `'markup'` 改为 `'code'` 可能影响其他依赖 `fileCategory` 的逻辑 | 低 | 需全局搜索 `fileCategory` 引用，确认无副作用 |
