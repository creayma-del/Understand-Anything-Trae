# P0.1: Vue SFC 详细设计规格

> 版本: 4.1
> 日期: 2026-06-04
> 状态: 已评审（修复 Critical 问题后）
> 变更: v4.0 方案升级 — 从"无 Tree-sitter 模式"升级为"独立 Plugin + 官方编译器"方案，使用 @vue/compiler-sfc 解析 SFC 结构 + TypeScript tree-sitter 解析 script 内容，实现确定性结构化分析

## 1. 目标

为项目新增 Vue Single File Component (`.vue`) 的完整支持，包括：
- 语言配置注册（core 包 + scan-project.mjs 双端）
- **确定性结构化提取**（@vue/compiler-sfc + TypeScript tree-sitter）
- LLM 分析提示词
- 导入解析增强
- Vue 框架配置联动
- 指纹系统升级（从降级模式升级为结构化分析）

## 2. Vue SFC 文件结构分析

Vue SFC 是一种混合语言文件，包含三个顶级块：

```vue
<template>
  <div class="app">
    <ChildComponent :prop="value" @event="handler" />
  </div>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue'
import ChildComponent from './ChildComponent.vue'

const value = ref('')
const handler = () => { /* ... */ }
</script>

<style scoped>
.app { display: flex; }
</style>
```

### 2.1 提取策略：官方编译器 + TypeScript tree-sitter

**方案升级说明**: 之前因 tree-sitter-vue WASM 不可用而降级为纯 LLM 分析。但纯 LLM 分析存在不可靠、不全面、不可复现的风险。现升级为使用 Vue 官方编译器 `@vue/compiler-sfc` 解析 SFC 结构，再用 TypeScript tree-sitter 解析 script 内容，实现**确定性**结构化分析。

```
阶段 1: @vue/compiler-sfc.parse() 解析 .vue 文件
  → 获取 SFCDescriptor：script / scriptSetup / template / styles
  → 提取 script 内容（loc.startLine 用于行号偏移）
  → 提取 template 中的组件引用（PascalCase 标签）

阶段 2: TypeScript tree-sitter 解析 script 内容
  → 复用 TreeSitterPlugin 中已加载的 TypeScript grammar
  → 复用 TypeScriptExtractor 提取函数/类/导入/导出/调用图
  → 行号偏移修正：script 节点在 .vue 文件中的起始行
```

### 2.2 WASM 兼容性验证结论（保留参考）

tree-sitter-vue 的 WASM grammar 不可用（external scanner 限制），因此：
- LanguageConfig 中**不设 `treeSitter` 字段**（避免 TreeSitterPlugin 尝试加载 WASM）
- 不创建 LanguageExtractor（不依赖 tree-sitter-vue AST）
- **改为创建独立的 VueSfcPlugin**，使用 @vue/compiler-sfc 替代 tree-sitter-vue

## 4. Layer 2: 结构化提取 — VueSfcPlugin

### 4.1 架构决策：独立 AnalyzerPlugin

**不使用 TreeSitterPlugin 处理 .vue 文件**，而是创建独立的 `VueSfcPlugin`，原因：

1. TreeSitterPlugin 要求 LanguageConfig 包含 `treeSitter` 字段才能加载 grammar
2. tree-sitter-vue WASM 不可用（external scanner 限制）
3. Vue SFC 需要先拆分 SFC 结构，再解析 script 内容，这是 TreeSitterPlugin 不支持的两阶段流程
4. 独立插件遵循项目已有的插件模式（MarkdownParser、YAMLConfigParser 等都是独立插件）

### 4.2 VueSfcPlugin 设计

**文件**: `packages/core/src/plugins/parsers/vue-sfc-parser.ts`

```typescript
import type { AnalyzerPlugin, StructuralAnalysis, ImportResolution, CallGraphEntry } from "../../types.js";

/**
 * Vue SFC parser plugin.
 *
 * Uses @vue/compiler-sfc to parse .vue files, then delegates script
 * content to TypeScript tree-sitter for structural analysis.
 * Template component references are extracted via the compiler's AST.
 */
export class VueSfcPlugin implements AnalyzerPlugin {
  readonly name = "vue-sfc-parser";
  readonly languages = ["vue-sfc"];

  // Dependencies injected via init()
  private tsPlugin: TreeSitterPlugin | null = null;

  /**
   * Initialize with a reference to the TreeSitterPlugin for
   * TypeScript grammar access.
   */
  init(tsPlugin: TreeSitterPlugin): void {
    this.tsPlugin = tsPlugin;
  }

  analyzeFile(filePath: string, content: string): StructuralAnalysis {
    // 1. Parse SFC with @vue/compiler-sfc
    // 2. Extract script content + line offset
    // 3. [降级守卫] 如果 tsPlugin 未初始化，仅返回 template 组件引用
    // 4. Delegate to TypeScript tree-sitter for script analysis
    // 5. Extract template component references
    // 6. Merge results with line offset correction
  }

  resolveImports(filePath: string, content: string): ImportResolution[] {
    // Delegate to analyzeFile().imports, then resolve paths
  }

  extractCallGraph(filePath: string, content: string): CallGraphEntry[] {
    // Delegate to TypeScript tree-sitter for script call graph
    // Apply line offset correction
  }
}
```

### 4.3 @vue/compiler-sfc 解析流程

```typescript
import { parse as parseSfc } from "@vue/compiler-sfc";

function parseVueSfc(content: string, filePath: string) {
  const descriptor = parseSfc(content, {
    filename: filePath,
    // 不需要 sourceMap
  });

  // descriptor 结构:
  // - script: { content, lang, loc: { start: { line, column }, end: { line, column } } } | null
  // - scriptSetup: { content, lang, loc: { ... } } | null
  // - template: { content, ast, loc: { ... } } | null
  // - styles: Array<{ content, scoped, module, lang, loc: { ... } }>

  // 提取 script 内容（优先 scriptSetup，其次 script）
  const scriptBlock = descriptor.scriptSetup ?? descriptor.script;
  const scriptContent = scriptBlock?.content ?? "";
  const scriptLang = scriptBlock?.lang ?? "js";
  const scriptStartLine = scriptBlock?.loc.start.line ?? 1;

  // 提取 template 中的组件引用
  const templateComponents = extractTemplateComponents(descriptor.template);

  return { scriptContent, scriptLang, scriptStartLine, templateComponents, descriptor };
}
```

### 4.4 Script 内容的 TypeScript tree-sitter 解析

```typescript
analyzeFile(filePath: string, content: string): StructuralAnalysis {
  const { scriptContent, scriptLang, scriptStartLine, templateComponents } =
    parseVueSfc(content, filePath);

  // 如果没有 script 块，仅返回 template 组件引用
  if (!scriptContent) {
    return {
      functions: [],
      classes: [],
      imports: templateComponents.map(comp => ({
        source: comp.name,  // PascalCase 组件名
        specifiers: [comp.name],
        lineNumber: comp.lineNumber,
      })),
      exports: [],
    };
  }

  // 使用 TypeScript tree-sitter 解析 script 内容
  // [降级守卫] tsPlugin 未初始化时返回空分析
  if (!this.tsPlugin) {
    return {
      functions: [],
      classes: [],
      imports: templateComponents.map(comp => ({
        source: comp.name,
        specifiers: [comp.name],
        lineNumber: comp.lineNumber,
      })),
      exports: [],
    };
  }

  const tsAnalysis = this.tsPlugin.analyzeFile(
    // 传入虚拟路径以触发 TS/JS 语法选择
    scriptLang === "ts" ? "virtual.ts" : "virtual.js",
    scriptContent,
  );

  // 行号偏移修正：script 在 .vue 文件中的起始行
  const lineOffset = scriptStartLine - 1; // compiler-sfc 的行号从 1 开始
  const correctedFunctions = tsAnalysis.functions.map(fn => ({
    ...fn,
    lineRange: [fn.lineRange[0] + lineOffset, fn.lineRange[1] + lineOffset] as [number, number],
  }));
  const correctedClasses = tsAnalysis.classes.map(cls => ({
    ...cls,
    lineRange: [cls.lineRange[0] + lineOffset, cls.lineRange[1] + lineOffset] as [number, number],
  }));
  const correctedImports = tsAnalysis.imports.map(imp => ({
    ...imp,
    lineNumber: imp.lineNumber + lineOffset,
  }));
  const correctedExports = tsAnalysis.exports.map(exp => ({
    ...exp,
    lineNumber: exp.lineNumber + lineOffset,
  }));

  // 合并 template 组件引用到 imports
  // （仅补充 script 中未 import 的组件引用）
  // 去重：检查组件名是否已出现在 import specifiers 中
  const importedSpecifiers = new Set(correctedImports.flatMap(i => i.specifiers));
  const templateImports = templateComponents
    .filter(comp => !importedSpecifiers.has(comp.name))
    .map(comp => ({
      source: comp.name,
      specifiers: [comp.name],
      lineNumber: comp.lineNumber,
    }));

  return {
    functions: correctedFunctions,
    classes: correctedClasses,
    imports: [...correctedImports, ...templateImports],
    exports: correctedExports,
  };
}
```

### 4.5 Template 组件引用提取

```typescript
interface TemplateComponentRef {
  name: string;       // PascalCase 组件名，如 "ChildComponent"
  lineNumber: number; // 在 .vue 文件中的行号
}

/**
 * 从 template AST 中提取 PascalCase 组件引用。
 * Vue 约定 PascalCase 标签为组件，小写标签为 HTML 元素。
 */
function extractTemplateComponents(
  template: { ast: any } | null,
): TemplateComponentRef[] {
  if (!template?.ast) return [];

  const components: TemplateComponentRef[] = [];
  const isPascalCase = (name: string) => /^[A-Z]/.test(name);

  function walk(node: any) {
    if (node.type === 1 /* NodeTypes.ELEMENT */) {
      if (isPascalCase(node.tag)) {
        components.push({
          name: node.tag,
          lineNumber: node.loc.start.line,
        });
      }
      // 递归子节点
      if (node.children) {
        for (const child of node.children) {
          walk(child);
        }
      }
    } else if (node.type === 0 /* NodeTypes.ROOT */ ||
               node.type === 2 /* NodeTypes.TEXT */ ||
               node.type === 5 /* NodeTypes.INTERPOLATION */) {
      // 跳过非元素节点
      if (node.children) {
        for (const child of node.children) {
          walk(child);
        }
      }
    }
  }

  walk(template.ast);
  return components;
}
```

### 4.6 Call Graph 提取

```typescript
extractCallGraph(filePath: string, content: string): CallGraphEntry[] {
  const { scriptContent, scriptLang, scriptStartLine } = parseVueSfc(content, filePath);
  if (!scriptContent || !this.tsPlugin) return [];

  const callGraph = this.tsPlugin.extractCallGraph(
    scriptLang === "ts" ? "virtual.ts" : "virtual.js",
    scriptContent,
  );

  // 行号偏移修正
  const lineOffset = scriptStartLine - 1;
  return callGraph.map(entry => ({
    ...entry,
    lineNumber: entry.lineNumber + lineOffset,
  }));
}
```

### 4.10 与 TreeSitterPlugin 的关系

| 维度 | TreeSitterPlugin | VueSfcPlugin |
|---|---|---|
| 处理文件 | .ts/.tsx/.js/.jsx/.mjs/.cjs/.py | .vue |
| 解析方式 | tree-sitter WASM | @vue/compiler-sfc + TS tree-sitter |
| 语言注册 | LanguageConfig.treeSitter | LanguageConfig（无 treeSitter） |
| 插件注册 | PluginRegistry.register() | PluginRegistry.register() |
| 依赖关系 | 无 | 依赖 TreeSitterPlugin 的 TS grammar |
| 行号修正 | 不需要 | 需要（script 在 .vue 中的偏移） |

### 5.1 文件: `skills/understand/languages/vue-sfc.md`

```markdown
# Vue SFC Language Prompt Snippet

## Key Concepts

- **Single-File Components**: `.vue` files encapsulate template, script, and style in one file
- **Composition API**: `setup()` function and `<script setup>` syntax with composables
- **Options API**: `data()`, `methods`, `computed`, `watch` object-based API
- **Compiler Macros**: `defineProps`, `defineEmits`, `defineExpose`, `defineModel`, `withDefaults`
- **Reactivity**: `ref()`, `reactive()`, `computed()`, `watch()`, `watchEffect()`
- **Component Communication**: props down, events up, provide/inject for deep passing
- **Scoped Styles**: `<style scoped>` limits CSS to the component; `<style module>` uses CSS Modules
- **v-bind in CSS**: `v-bind(property)` to use reactive values in CSS
- **Slots**: Default slots, named slots, scoped slots for content distribution
- **Directives**: v-model, v-if/v-else/v-else-if, v-for, v-show, v-slot, v-once, v-memo

## Notable File Patterns

- `*.vue` — Vue Single File Components
- `App.vue` — Root application component
- `*.spec.vue`, `*.test.vue` — Vue component test files
- `<script setup>` — Recommended concise syntax (Composition API)
- `<script>` + `setup()` — Explicit Composition API setup function
- `<script>` — Options API (legacy)

## Edge Patterns

- Vue SFC files are `contains` by the parent component that imports them
- `<script setup>` imports are `imports` edges to other modules
- Template component references (PascalCase tags) are `imports` edges to child components
- `defineProps` creates `depends_on` edges from the component to the prop type definitions
- `defineEmits` creates `depends_on` edges from the component to event type definitions
- `provide/inject` creates implicit `depends_on` edges across the component tree
- Scoped styles are `related` to the component they style
- Composable usage (`useX()`) creates `depends_on` edges to the composable module

## Summary Style

> "Vue SFC component implementing a search form with debounced input, emits search events to parent."
> "Root App.vue component mounting the top-level layout with router-view and global navigation."
> "Vue SFC with script setup defining reactive form state, computed validation, and API submission logic."
```

## 9. 与 v3.0 的关键差异

| 维度 | v3.0（无 Tree-sitter 模式） | v4.0（独立 Plugin + 官方编译器） |
|---|---|---|
| 结构化分析 | 纯 LLM，非确定性 | @vue/compiler-sfc + TS tree-sitter，确定性 |
| 新增依赖 | 无 | `@vue/compiler-sfc` |
| 新增文件 | 2（配置 + 指南） | 3（配置 + 插件 + 指南） |
| 修改文件 | 4 | 6 |
| 指纹系统 | 降级模式（hasStructuralAnalysis: false） | 结构化指纹（hasStructuralAnalysis: true） |
| 行号精度 | LLM 估算 | 精确到行（编译器提供） |
| template 组件引用 | LLM 识别 | 编译器 AST 提取，确定性 |
| call graph | 不支持 | 支持（TS tree-sitter 提取 + 行号偏移） |
