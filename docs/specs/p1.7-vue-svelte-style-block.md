# P1.7: Vue/Svelte Style 块分析 详细设计规格

> 版本: 1.0
> 日期: 2026-06-05
> 状态: 已评审（修复 HIGH 问题后）
> 变更: 初始版本 — 扩展 VueSfcPlugin 和 SveltePlugin，提取 `<style>` 块并委托 CssPlugin 进行 CSS 结构化分析，实现 Vue/Svelte 文件中样式块的确定性提取与行号修正

## 1. 目标与范围

为 VueSfcPlugin 和 SveltePlugin 新增 `<style>` 块的结构化分析能力，包括：

- **依赖注入扩展**：`registerAllParsers` 新增 `cssPlugin` 参数，VueSfcPlugin / SveltePlugin 的 `init()` 方法扩展为接受 `cssPlugin`
- **VueSfcPlugin 扩展**：提取 `SFCDescriptor.styles` 中的每个 style 块，委托 CssPlugin 分析，修正行号偏移，标记 scoped/module，识别 `v-bind()` in CSS
- **SveltePlugin 扩展**：提取 `ast.css` 中的 style 块，委托 CssPlugin 分析，字符偏移转行号，标记 scoped（Svelte style 默认 scoped）
- **mergeStyleAnalysis() 辅助方法**：统一合并 style 分析结果到主 StructuralAnalysis，处理行号偏移、scoped/module 标记、v-bind() 识别
- **注册变更**：`parsers/index.ts` 中 `registerAllParsers` 签名扩展

**前置依赖**：P1.1（CssPlugin）必须已完成。

**范围边界**：

- ✅ Vue `<style scoped>` / `<style module>` / `<style lang="scss">` 的结构化提取
- ✅ Svelte `<style>` 的结构化提取（默认 scoped）
- ✅ Vue `v-bind()` in CSS 的识别与 `depends_on` 边生成
- ✅ 多 style 块合并
- ✅ 行号偏移修正
- ❌ CSS-in-JS 支持（P1.8 范围）
- ❌ Vue `<style>` 中的 CSS 预处理器运行时行为分析（如 Sass 编译后结果）
- ❌ Svelte `:global()` 选择器的特殊处理（仅作为普通 CSS 规则提取）

## 2. Vue/Svelte Style 块结构分析

### 2.1 Vue SFC 中的 Style 块

Vue SFC 支持多个 `<style>` 块，每个块可独立设置 `scoped`、`module`、`lang` 属性：

```vue
<template>
  <div class="app">
    <ChildComponent :prop="value" />
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'
const primaryColor = ref('#3366cc')
</script>

<style scoped>
.app {
  display: flex;
  color: v-bind(primaryColor);
}
</style>

<style module>
.btn {
  background: #fff;
}
</style>

<style lang="scss">
$font-size: 14px;
.container {
  font-size: $font-size;
}
</style>
```

`@vue/compiler-sfc` 解析后的 `SFCDescriptor.styles` 结构：

```typescript
descriptor.styles: SFCStyleBlock[]

interface SFCStyleBlock {
  content: string;                              // style 内容（不含标签）
  loc: { start: { line: number }, end: { line: number } };  // 行号定位
  scoped: boolean;                              // 是否 <style scoped>
  module: string | boolean | null;              // 是否 <style module>（可为自定义模块名）
  lang?: string;                                // 预处理语言（scss, sass 等）
}
```

### 2.2 Svelte 中的 Style 块

Svelte 文件仅支持一个 `<style>` 块，且默认 scoped：

```svelte
<script>
  let count = 0;
</script>

<button class="btn" on:click={() => count++}>
  {count}
</button>

<style>
  .btn {
    padding: 8px 16px;
    background: #3366cc;
    color: white;
  }
</style>
```

`svelte/compiler` 解析后的 `ast.css` 结构：

```typescript
ast.css: {
  content: string;    // style 内容（不含标签）
  start: number;      // 字符偏移（含 <style> 标签起始位置）
  end: number;        // 字符偏移
} | null
```

**关键差异**：Svelte 的 `css.start` 是字符偏移而非行号，需要转换为行号。且 Svelte `<style>` 始终 scoped，无需额外标记。

### 2.3 提取策略：委托 CssPlugin

```
阶段 1: 从 SFC/Svelte 编译器提取 style 块
  → Vue: 遍历 descriptor.styles[]，获取 content + loc.start.line + scoped/module/lang
  → Svelte: 从 ast.css 获取 content + start（字符偏移）

阶段 2: 委托 CssPlugin.analyzeFile() 分析 style 内容
  → 传入虚拟路径（如 `${filePath}#style`）触发 CSS/SCSS 语法选择
  → 获取 StructuralAnalysis（含 cssRules、imports、exports）

阶段 3: 行号偏移修正 + 标记合并
  → Vue: styleBlock.loc.start.line 作为偏移量
  → Svelte: content.slice(0, ast.css.start).split('\n').length 作为偏移量
  → 对 cssRules.lineRange、imports.lineNumber、exports.lineNumber 应用偏移
  → 添加 scoped-style / css-modules 标签
  → 识别 v-bind() in CSS → depends_on 边
```

### 4.3 analyzeFile() 中提取 style 块

在现有 `analyzeFile()` 返回结果前，新增 style 块提取逻辑：

```typescript
analyzeFile(filePath: string, content: string): StructuralAnalysis {
  const { scriptContent, scriptLang, scriptStartLine, templateComponents, descriptor } =
    parseVueSfc(content, filePath);

  // ... 现有 script + template 分析逻辑 ...

  const result: StructuralAnalysis = {
    functions: correctedFunctions,
    classes: correctedClasses,
    imports: [...correctedImports, ...templateImports],
    exports: correctedExports,
  };

  // ↓↓↓ 新增：style 块分析 ↓↓↓
  if (this.cssPlugin && descriptor.styles.length > 0) {
    for (const styleBlock of descriptor.styles) {
      const styleResult = this.cssPlugin.analyzeFile(
        `${filePath}#style`,  // 虚拟路径，触发 CSS/SCSS 语法选择
        styleBlock.content,
      );

      // 行号偏移修正：style 块在 .vue 文件中的起始行
      // styleBlock.loc.start.line 是 <style> 标签所在行号（1-based）
      // CssPlugin 返回的行号基于 styleBlock.content（从第 1 行开始）
      // .vue 文件实际行号 = CssPlugin 行号 + styleBlock.loc.start.line
      // 详见 4.4 节精确计算说明
      const styleOffset = styleBlock.loc.start.line;

      // 合并 style 分析结果到主结果
      this.mergeStyleAnalysis(result, styleResult, styleOffset, {
        scoped: styleBlock.scoped,
        module: styleBlock.module != null,
      });
    }
  }

  return result;
}
```

### 4.4 行号偏移修正

Vue `@vue/compiler-sfc` 提供的 `styleBlock.loc.start.line` 是基于 1 的行号，表示 `<style>` 标签所在行。style 块内容的实际起始行需要进一步计算：

```typescript
// styleBlock.loc.start.line 是 <style> 标签的行号
// style 内容从 <style> 标签下一行开始
// 但 CssPlugin 返回的行号是基于 styleBlock.content 的（从第 1 行开始）
// 因此偏移量 = styleBlock.loc.start.line（<style> 标签行号）
// CssPlugin 行号 1 → .vue 文件行号 = 1 + styleBlock.loc.start.line = styleBlock.loc.start.line + 1
// 但实际上 <style> 标签本身占一行，内容从 loc.start.line + 1 行开始
// 更精确的做法：计算 style 内容在 .vue 文件中的实际起始行

// 简化方案：使用 styleBlock.loc.start.line 作为偏移基准
// CssPlugin 返回的 lineRange 基于 styleBlock.content（从第 1 行开始）
// .vue 文件中的实际行号 = CssPlugin 行号 + (styleBlock.loc.start.line)
// 因为 loc.start.line 是 <style> 标签所在行，内容从下一行开始
// 所以偏移量 = styleBlock.loc.start.line（<style> 标签行号即为偏移量）
const styleOffset = styleBlock.loc.start.line;
```

**精确计算说明**：

`@vue/compiler-sfc` 的 `SFCStyleBlock.loc.start.line` 指向 `<style>` 标签所在行。style 块内容从 `<style>` 标签的下一行开始。CssPlugin 解析 `styleBlock.content` 时，内容的第一行对应行号 1。因此：

```
.vue 文件实际行号 = CssPlugin 行号 + styleBlock.loc.start.line
```

例如：`<style scoped>` 在第 20 行，style 内容第一行在 .vue 文件中是第 21 行。CssPlugin 返回行号 1，`1 + 20 = 21`，正确。

### 4.5 scoped/module 标记

当 style 块为 `<style scoped>` 或 `<style module>` 时，需要在合并的 `cssRules` 中添加标签信息。通过在 `CssRuleInfo` 的 `tags` 字段中添加标记实现（`tags` 字段为 P1.7 对 P1.1 `CssRuleInfo` 的扩展，详见 4.7 节）：

```typescript
interface StyleBlockMeta {
  scoped: boolean;
  module: boolean;
}

mergeStyleAnalysis(
  result: StructuralAnalysis,
  styleResult: StructuralAnalysis,
  lineOffset: number,
  meta: StyleBlockMeta,
): void {
  // ... 行号偏移修正 ...

  // 添加 scoped/module 标记到 cssRules
  for (const rule of styleResult.cssRules ?? []) {
    const tags: string[] = [...(rule.tags ?? [])];
    if (meta.scoped) tags.push("scoped-style");
    if (meta.module) tags.push("css-modules");

    const correctedRule = {
      ...rule,
      lineRange: [rule.lineRange[0] + lineOffset, rule.lineRange[1] + lineOffset] as [number, number],
      // 使用 tags 字段存放语义标签，而非 declarations
      tags,
    };
    result.cssRules!.push(correctedRule);
  }
}
```

### 4.6 v-bind() in CSS 识别

Vue 3.2+ 支持 `v-bind()` 在 CSS 中使用响应式变量。需要在 style 内容中识别此模式并生成 `depends_on` 边。

```typescript
/**
 * 识别 Vue v-bind() in CSS 模式。
 * 示例：color: v-bind(primaryColor);
 * 生成 depends_on 边：style 规则 → script 中的响应式变量
 */
function extractVBindReferences(
  styleContent: string,
  styleOffset: number,
): Array<{ variable: string; lineNumber: number }> {
  const references: Array<{ variable: string; lineNumber: number }> = [];
  const vBindRegex = /v-bind\s*\(\s*([a-zA-Z_$][a-zA-Z0-9_$]*)\s*\)/g;

  const lines = styleContent.split('\n');
  for (let i = 0; i < lines.length; i++) {
    const line = lines[i];
    let match: RegExpExecArray | null;
    while ((match = vBindRegex.exec(line)) !== null) {
      references.push({
        variable: match[1],
        lineNumber: i + 1 + styleOffset,  // 1-based + 偏移
      });
    }
    vBindRegex.lastIndex = 0;  // 重置正则状态
  }

  return references;
}
```

v-bind() 引用信息将添加到 `StructuralAnalysis.imports` 中，`source` 为变量名，`specifiers` 包含 `"v-bind"` 标记：

```typescript
// 在 mergeStyleAnalysis() 中
const vBindRefs = extractVBindReferences(styleBlock.content, styleOffset);
for (const ref of vBindRefs) {
  result.imports.push({
    source: ref.variable,
    specifiers: ["v-bind"],
    lineNumber: ref.lineNumber,
  });
}
```

### 5.3 analyzeFile() 中提取 style 块

在现有 `analyzeFile()` 返回结果前，新增 style 块提取逻辑：

```typescript
analyzeFile(filePath: string, content: string): StructuralAnalysis {
  const { scriptBlocks, templateComponents, ast } =
    parseSvelteSfc(content, filePath);

  // ... 现有 script + template 分析逻辑 ...

  const result: StructuralAnalysis = {
    functions: allFunctions,
    classes: allClasses,
    imports: [...allImports, ...templateImports],
    exports: allExports,
  };

  // ↓↓↓ 新增：style 块分析 ↓↓↓
  if (this.cssPlugin && ast.css) {
    // Svelte 的 ast.css.content 是 style 块的纯文本内容
    // 需要从原始文件内容中提取（与 script 块相同的策略）
    const styleContent = extractStyleContent(content, ast.css);
    const styleResult = this.cssPlugin.analyzeFile(
      `${filePath}#style`,
      styleContent,
    );

    // 字符偏移转行号
    // ast.css.start 是 <style> 标签的字符偏移位置
    // 偏移量 = <style> 标签所在行号（1-based），与 Vue 的 styleOffset 语义一致
    const styleOffset = content.slice(0, ast.css.start).split('\n').length;

    // Svelte <style> 始终 scoped
    this.mergeStyleAnalysis(result, styleResult, styleOffset, {
      scoped: true,
      module: false,
    });
  }

  return result;
}
```

### 5.4 字符偏移转行号

Svelte 编译器提供的是字符偏移（`ast.css.start`），需要转换为行号：

```typescript
// ast.css.start 是 <style> 标签起始位置的字符偏移
// 转换为行号：计算从文件开头到该偏移位置的换行符数量
// 结果为 <style> 标签所在行号（1-based），作为偏移量
// CssPlugin 返回的行号基于 styleContent（从第 1 行开始）
// .svelte 文件中的实际行号 = CssPlugin 行号 + styleOffset
const styleOffset = content.slice(0, ast.css.start).split('\n').length;
```

**计算说明**：

- `content.slice(0, ast.css.start)` 获取 `<style>` 标签之前的所有内容
- `.split('\n').length` 计算行数（即 `<style>` 标签所在行号，从 1 开始）
- CssPlugin 返回的行号基于 styleContent（从第 1 行开始）
- .svelte 文件中的实际行号 = CssPlugin 行号 + styleOffset

例如：`<style>` 在第 15 行，style 内容第一行在 .svelte 文件中是第 16 行。CssPlugin 返回行号 1，`1 + 15 = 16`，正确。

### 5.5 extractStyleContent 辅助函数

```typescript
/**
 * 从原始文件内容中提取 Svelte style 块的纯文本。
 *
 * Svelte 5 的 ast.css 不直接提供 content 字符串，
 * 需要从原始内容中按 start/end 偏移量提取。
 * ast.css.start/end 覆盖整个 <style> 元素（含标签），
 * 需要跳过 <style...> 开始标签和 </style> 结束标签。
 */
function extractStyleContent(
  fileContent: string,
  cssBlock: { start: number; end: number },
): string {
  // 提取整个 <style>...</style> 块
  const rawBlock = fileContent.slice(cssBlock.start, cssBlock.end);

  // 去除 <style> 开始标签和 </style> 结束标签
  const contentMatch = rawBlock.match(/^<style[^>]*>\n?([\s\S]*?)\n?<\/style>$/);
  return contentMatch ? contentMatch[1] : rawBlock;
}
```

### 5.6 scoped 标记

Svelte 的 `<style>` 块始终 scoped，编译器会自动添加唯一的 scoped 属性（如 `svelte-xxx`）。在 `mergeStyleAnalysis()` 中传入 `scoped: true`：

```typescript
this.mergeStyleAnalysis(result, styleResult, styleOffset, {
  scoped: true,   // Svelte style 默认 scoped

```

## 6. mergeStyleAnalysis() 辅助方法

### 6.1 方法签名

```typescript
/**
 * Style 块元信息，用于标记 scoped/module。
 */
interface StyleBlockMeta {
  scoped: boolean;
  module: boolean;
}

/**
 * 将 CssPlugin 对 style 块的分析结果合并到主 StructuralAnalysis 中。
 *
 * 处理：
 * 1. cssRules 行号偏移修正 + scoped/module 标记
 * 2. imports 行号偏移修正
 * 3. exports 行号偏移修正
 * 4. v-bind() in CSS 识别（仅 Vue）
 */
private mergeStyleAnalysis(
  result: StructuralAnalysis,
  styleResult: StructuralAnalysis,
  lineOffset: number,
  meta: StyleBlockMeta,
): void
```

### 6.2 完整实现

```typescript
private mergeStyleAnalysis(
  result: StructuralAnalysis,
  styleResult: StructuralAnalysis,
  lineOffset: number,
  meta: StyleBlockMeta,
): void {
  // 确保 cssRules 数组存在
  if (!result.cssRules) {
    result.cssRules = [];
  }

  // 1. 合并 cssRules（行号偏移 + 标签标记）
  for (const rule of styleResult.cssRules ?? []) {
    const tags: string[] = [...(rule.tags ?? [])];
    if (meta.scoped) tags.push("scoped-style");
    if (meta.module) tags.push("css-modules");

    result.cssRules.push({
      ...rule,
      lineRange: [
        rule.lineRange[0] + lineOffset,
        rule.lineRange[1] + lineOffset,
      ] as [number, number],
      tags,
    });
  }

  // 2. 合并 imports（行号偏移）
  for (const imp of styleResult.imports) {
    result.imports.push({
      ...imp,
      lineNumber: imp.lineNumber + lineOffset,
    });
  }

  // 3. 合并 exports（行号偏移）
  for (const exp of styleResult.exports) {
    result.exports.push({
      ...exp,
      lineNumber: exp.lineNumber + lineOffset,
    });
  }
}
```

### 6.3 v-bind() 处理位置

v-bind() in CSS 的识别在 VueSfcPlugin 的 `analyzeFile()` 中调用 `mergeStyleAnalysis()` 之前执行，将 v-bind 引用作为额外 imports 添加到 styleResult 中，然后统一合并：

```typescript
// VueSfcPlugin.analyzeFile() 中
if (this.cssPlugin && descriptor.styles.length > 0) {
  for (const styleBlock of descriptor.styles) {
    const styleResult = this.cssPlugin.analyzeFile(
      `${filePath}#style`,
      styleBlock.content,
    );

    const styleOffset = styleBlock.loc.start.line;

    // 识别 v-bind() in CSS
    const vBindRefs = extractVBindReferences(styleBlock.content, styleOffset);
    for (const ref of vBindRefs) {
      styleResult.imports.push({
        source: ref.variable,
        specifiers: ["v-bind"],
        lineNumber: ref.lineNumber,
      });
    }

    this.mergeStyleAnalysis(result, styleResult, styleOffset, {
      scoped: styleBlock.scoped,
      module: styleBlock.module != null,
    });
  }
}
```

## 11. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|---|---|---|---|
| Vue style 块行号偏移计算不准确 | cssRules 行号与源文件不一致 | 中 | 单元测试 STYLE-006 专门验证行号偏移；使用 `@vue/compiler-sfc` 提供的 `loc.start.line` 作为权威来源 |
| Svelte 字符偏移转行号精度 | `ast.css.start` 字符偏移转行号存在 ±1 误差 | 中 | 单元测试 STYLE-004 验证行号；使用 `content.slice(0, offset).split('\n').length` 标准转换方式 |
| Svelte `ast.css` 内容提取方式变更 | Svelte 5 编译器版本差异导致 `ast.css` 结构不同 | 低 | `extractStyleContent()` 使用正则匹配标签边界，兼容不同版本；降级时返回原始块内容 |
| `mergeStyleAnalysis()` 修改 result 的副作用 | 直接修改传入的 result 对象可能影响调用方 | 低 | 这是设计意图（就地合并），与现有 VueSfcPlugin 中合并 template 组件引用的模式一致 |
| v-bind() 正则匹配遗漏 | 复杂的 v-bind 表达式（如 `v-bind('primary-color')`）可能未被识别 | 低 | 当前仅支持简单标识符形式 `v-bind(variableName)`；kebab-case 形式可在后续迭代支持 |
| `registerAllParsers` 签名扩展影响现有调用方 | 新增可选参数不破坏现有调用，但调用方可能忘记传入 cssPlugin | 低 | 参数为可选，不传时功能降级但不报错；文档中明确建议传入 cssPlugin |
| CssPlugin 解析 style 内容失败 | style 块含无效 CSS 语法时 CssPlugin 返回空分析 | 低 | CssPlugin 已有降级处理（P1.1），返回空 StructuralAnalysis，不影响主分析流程 |
