## 1. 目标与范围

### 1.1 问题

当前三个确定性脚本各自独立初始化 tree-sitter 并独立读取/解析相同的项目文件：

1. **`extract-import-map.mjs`** — 提取导入关系（初始化 tree-sitter → 解析所有文件 → `importMap`）
2. **`compute-batches.mjs`** 的 `extractExports()` — 提取导出符号（初始化 tree-sitter → 解析所有文件 → `exportsMap`）
3. **`extract-all-structure.mjs`**（P2.3 新增）— 提取结构数据（初始化 tree-sitter → 解析所有文件 → `structure`）

对于 300 文件项目，这意味着：
- **3 次 tree-sitter 初始化**（单次 ~260ms，含 WASM 运行时 + 10 种语言语法加载）
- **3 次文件读取**（每个文件被 `readFileSync` 3 次）
- **3 次文件解析**（每个文件被 `registry.analyzeFile()` 3 次）

### 1.2 目标

将三个脚本的核心逻辑合并为 `analyze-project.mjs`，实现：
- **1 次 tree-sitter 初始化**（3 → 1）
- **1 次文件读取 + 解析**（3 → 1）
- **同时产出 3 个输出文件**：`import-map.json`、`structure-results.json`、`exports-map.json`
- 300 文件项目节省 ~6-30s（初始化 + 重复 I/O + 重复解析）

### 1.3 范围

**包含**：
- 新增 `analyze-project.mjs` 统一分析脚本
- 修改 `compute-batches.mjs` 的 `extractExports()` 优先读取 `exports-map.json`
- 修改 `SKILL.md` Phase 1 / Phase 1.7 流程
- 新增测试文件 `test_analyze_project.test.mjs`
- 更新 `test_compute_batches.test.mjs` 新增 CB-005 / CB-006

**不包含**：
- 删除 `extract-import-map.mjs`（保留为独立入口，供增量更新和独立调试使用）
- 删除 `extract-structure.mjs`（保留为独立入口，供增量更新和独立调试使用）
- 修改 `scan-project.mjs`（文件枚举逻辑不变）
- 修改 `build-fingerprints.mjs`（指纹生成逻辑不变）
- 修改 core 包公共 API（PluginRegistry、TreeSitterPlugin 等接口不变）

### 1.4 约束

- 所有输出必须与独立脚本运行结果等价（或更优）
- 不引入新的外部依赖
- 不修改 core 包的公共 API
- 保持增量更新路径完全兼容（原脚本保留为独立入口）
- 新脚本遵循现有模式：CLI 参数、stderr 日志、JSON 输出、`isCliEntry()` 守卫
- 单文件错误不中断整体流程（per-file resilience）

## 2. 现状分析

### 2.1 三个脚本的重复操作

| 操作 | extract-import-map.mjs | compute-batches.mjs (extractExports) | extract-all-structure.mjs |
|------|----------------------|--------------------------------------|--------------------------|
| 初始化 TreeSitterPlugin | ✅ | ✅ | ✅ |
| 初始化 PluginRegistry | ✅ | ✅ | ✅ |
| 注册 CssPlugin | ✅ | ✅ | ✅ |
| 读取文件内容 | ✅ | ✅ | ✅ |
| 调用 `registry.analyzeFile()` | ✅ | ✅ | ✅ |
| 调用 `registry.extractCallGraph()` | ❌ | ❌ | ✅ |
| 产出 importMap | ✅ | ❌ | ❌ |
| 产出 exportsMap | ❌ | ✅ | ❌ |
| 产出 structure | ❌ | ❌ | ✅ |

**关键观察**：`registry.analyzeFile()` 是三个脚本共用的核心调用，其返回的 `StructuralAnalysis` 同时包含 `imports`、`exports`、`functions`、`classes` 等所有信息。三个脚本各自调用 `analyzeFile()` 后只使用了其中一部分数据，造成了 2/3 的计算浪费。

### 2.2 extract-import-map.mjs 的导入解析逻辑

`extract-import-map.mjs` 的核心逻辑分为两部分：

**Part A — tree-sitter 分析**（第 641-656 行）：
```javascript
const tsPlugin = new TreeSitterPlugin(tsConfigs);
await tsPlugin.init();
registry = new PluginRegistry();
registry.register(tsPlugin);
const cssPlugin = new CssPlugin();
registerAllParsers(registry, tsPlugin, cssPlugin);
```

**Part B — 导入路径解析**（第 309-598 行）：
- `resolveTsJsImport()` — TS/JS 相对导入 + tsconfig 路径别名
- `resolveCssImport()` — CSS/SCSS `@import`/`@use`/`@forward` + SCSS partial 探测
- `resolveHtmlImport()` — HTML `<script src>`/`<link href>`
- `extractRequireSources()` — CJS `require()` 补充扫描
- `resolveImport()` — 语言分发器

Part B 是纯函数逻辑（不依赖 tree-sitter），可直接提取复用。

### 2.3 compute-batches.mjs 的 extractExports() 逻辑

`extractExports()` 函数（第 42-89 行）：
1. 初始化 tree-sitter（与 extract-import-map.mjs 完全相同）
2. 遍历所有 code 文件
3. 调用 `registry.analyzeFile(file.path, content)` 获取 `analysis`
4. 从 `analysis.exports` 提取 `name` → `exportsByPath`

**核心发现**：`extractExports()` 仅使用了 `analysis.exports`，而 `analyzeFile()` 返回的 `analysis` 还包含 `imports`、`functions`、`classes` 等信息，这些信息被完全丢弃。

### 2.4 extract-all-structure.mjs 的结构提取逻辑

`extract-all-structure.mjs`（P2.3 新增）：
1. 初始化 tree-sitter
2. 遍历所有文件
3. 调用 `registry.analyzeFile()` + `registry.extractCallGraph()`
4. 通过 `buildResult()` 构建结构化输出

`buildResult()` 函数（来自 `extract-structure.mjs` 第 147-305 行）将 `StructuralAnalysis` 映射为输出 schema，包括 `functions`、`classes`、`exports`、`callGraph`、`metrics` 等。

## 3. 详细设计

### 3.1 新增脚本：analyze-project.mjs

**文件路径**：`skills/understand/analyze-project.mjs`

**CLI 接口**：

```bash
node analyze-project.mjs <projectRoot> <outputDir>
```

- `<projectRoot>`：项目根目录的绝对路径
- `<outputDir>`：中间文件输出目录（通常为 `$PROJECT_ROOT/.understand-anything-trae/intermediate`）

**输入**：`<outputDir>/scan-result.json`（Phase 0/1 产出）

**输出**：3 个 JSON 文件写入 `<outputDir>/`

| 输出文件 | 内容 | 格式 |
|---------|------|------|
| `import-map.json` | 导入关系映射 | `{ scriptCompleted, stats, importMap }` |
| `structure-results.json` | 全量结构数据 | `{ scriptCompleted, filesAnalyzed, filesSkipped, results }` |
| `exports-map.json` | 导出符号映射 | `{ scriptCompleted, stats, exportsMap }` |

#### 3.1.1 主流程伪代码

```javascript
async function main() {
  const [,, projectRoot, outputDir] = process.argv;

  // 1. 读取 scan-result.json
  const scanResult = JSON.parse(readFileSync(join(outputDir, 'scan-result.json'), 'utf-8'));
  const { files } = scanResult;

  // 2. 初始化 tree-sitter（一次）
  let registry = null;
  let treeSitterReady = false;
  try {
    const tsConfigs = builtinLanguageConfigs.filter(c => c.treeSitter);
    const tsPlugin = new TreeSitterPlugin(tsConfigs);
    await tsPlugin.init();
    registry = new PluginRegistry();
    registry.register(tsPlugin);
    const cssPlugin = new CssPlugin();
    registerAllParsers(registry, tsPlugin, cssPlugin);
    treeSitterReady = true;
  } catch (err) {
    process.stderr.write(
      `Warning: analyze-project: tree-sitter init failed (${err.message}) ` +
      `— all outputs will be empty\n`,
    );
  }

  // 3. 构建解析上下文（tsconfig 等，缓存一次）
  const ctx = buildResolutionContext(projectRoot, files);

  // 4. 遍历所有文件
  const importMap = {};
  const exportsMap = {};
  const structureResults = [];
  const filesSkipped = [];
  let filesWithImports = 0;
  let totalImportEdges = 0;
  let filesWithExports = 0;
  let totalExportSymbols = 0;

  for (const file of files) {
    const path = toPosix(file.path);

    // 非 code 文件（HTML 除外）跳过 tree-sitter 分析
    if (file.fileCategory !== 'code' && file.language !== 'html') {
      importMap[path] = [];
      exportsMap[path] = [];
      structureResults.push(buildBasicResult(file, 0, 0, null, null, null));
      continue;
    }

    // tree-sitter 初始化失败 → 空输出
    if (!treeSitterReady) {
      importMap[path] = [];
      exportsMap[path] = [];
      structureResults.push(buildBasicResult(file, 0, 0, null, null, null));
      continue;
    }

    // 读取文件内容（per-file resilience）
    const absolutePath = join(projectRoot, file.path);
    let content;
    try {
      content = readFileSync(absolutePath, 'utf-8');
    } catch (err) {
      process.stderr.write(
        `Warning: analyze-project: analysis failed for ${path} ` +
        `(read error: ${err.message})\n`,
      );
      importMap[path] = [];
      exportsMap[path] = [];
      filesSkipped.push(path);
      continue;
    }

    // 行数统计
    const lines = content.split('\n');
    const totalLines = content.endsWith('\n')
      ? Math.max(0, lines.length - 1) : lines.length;
    const nonEmptyLines = lines.filter(l => l.trim().length > 0).length;

    // 4a. 结构分析（一次 analyzeFile 调用）
    let analysis = null;
    try {
      analysis = registry.analyzeFile(file.path, content);
    } catch {
      // 分析失败 → 降级为基本指标
    }

    // 4b. 调用图提取（code 文件）
    let callGraph = null;
    if (file.fileCategory === 'code' || file.fileCategory === 'script') {
      try {
        const cg = registry.extractCallGraph(file.path, content);
        if (cg && cg.length > 0) {
          callGraph = cg.map(entry => ({
            caller: entry.caller,
            callee: entry.callee,
            lineNumber: entry.lineNumber,
          }));
        }
      } catch {
        // 调用图提取失败 — 非致命
      }
    }

    // 4c. 导入路径提取 → importMap
    // NOTE: resolveImports() 应从 shared/import-resolution.mjs 导入，
    // 该模块从 extract-import-map.mjs 提取的导入解析逻辑封装而成，
    // extract-import-map.mjs 和 analyze-project.mjs 均从该共享模块导入。
    const resolvedImports = resolveImports(registry, file, content, projectRoot, tsConfigPaths);
    importMap[path] = resolvedImports;
    if (resolvedImports.length > 0) {
      filesWithImports += 1;
      totalImportEdges += resolvedImports.length;
    }

    // 4d. 导出符号提取 → exportsMap
    const exports = (analysis?.exports || []).map(e => e.name).filter(Boolean);
    exportsMap[path] = exports;
    if (exports.length > 0) {
      filesWithExports += 1;
      totalExportSymbols += exports.length;
    }

    // 4e. 结构数据构建 → structureResults
    const batchImportData = importMap[path] || [];
    structureResults.push(buildResult(file, totalLines, nonEmptyLines, analysis, callGraph, batchImportData));
  }

  // 5. 写入三个输出文件
  writeFileSync(
    join(outputDir, 'import-map.json'),
    JSON.stringify({
      scriptCompleted: true,
      stats: { filesScanned: files.length, filesWithImports, totalEdges: totalImportEdges },
      importMap,
    }, null, 2),
    'utf-8',
  );

  writeFileSync(
    join(outputDir, 'structure-results.json'),
    JSON.stringify({
      scriptCompleted: true,
      filesAnalyzed: structureResults.length,
      filesSkipped,
      results: structureResults,
    }, null, 2),
    'utf-8',
  );

  writeFileSync(
    join(outputDir, 'exports-map.json'),
    JSON.stringify({
      scriptCompleted: true,
      stats: { filesScanned: files.length, filesWithExports, totalSymbols: totalExportSymbols },
      exportsMap,
    }, null, 2),
    'utf-8',
  );

  // 6. stderr 日志
  process.stderr.write(
    `analyze-project: filesScanned=${files.length} ` +
    `filesWithImports=${filesWithImports} totalImportEdges=${totalImportEdges} ` +
    `filesWithExports=${filesWithExports} totalExportSymbols=${totalExportSymbols}\n`,
  );
}
```

#### 3.1.2 导入解析函数（从共享模块导入）

导入解析逻辑应从 `extract-import-map.mjs` 提取到共享模块 `shared/import-resolution.mjs`，`extract-import-map.mjs` 和 `analyze-project.mjs` 均从该共享模块导入，避免代码重复。

**主流程中的调用方式**：

```javascript
import { resolveImports } from './shared/import-resolution.mjs';

// 在遍历文件时调用：
const resolvedImports = resolveImports(registry, file, content, projectRoot, tsConfigPaths);
```

**`resolveImports()` 内部逻辑**（合并了 extract-import-map.mjs 的两阶段逻辑）：

```javascript
/**
 * 解析单个文件的导入关系。
 *
 * 合并了 extract-import-map.mjs 的两阶段逻辑：
 * 1. 从 tree-sitter 分析结果的 analysis.imports 提取 ES module 导入
 * 2. 从 extractExtraImportSources() 补充 CJS require() 等额外来源
 *
 * @param {PluginRegistry} registry - 插件注册表
 * @param {Object} file - 文件元数据 { path, language, fileCategory }
 * @param {string} content - 文件内容
 * @param {string} projectRoot - 项目根目录
 * @param {Object} tsConfigPaths - tsconfig 路径别名配置
 * @returns {string[]} 已解析的项目内部路径列表（排序后）
 */
function resolveImports(registry, file, content, projectRoot, tsConfigPaths) {
  const resolvedSet = new Set();
  const analysis = registry.analyzeFile(file.path, content);
  const ctx = { projectRoot, fileSet: /* 从 scanResult 构建 */, tsConfigPaths };

  // 阶段 1：从 analysis.imports 提取 ES module 导入
  if (analysis?.imports) {
    for (const imp of analysis.imports) {
      const outs = resolveImport(imp, file, ctx);
      for (const out of outs) {
        if (out && ctx.fileSet.has(out)) {
          resolvedSet.add(out);
        }
      }
    }
  }

  // 阶段 2：补充 CJS require() 等额外来源
  for (const extra of extractExtraImportSources(file, content)) {
    const outs = resolveImport({ source: extra, specifiers: [] }, file, ctx);
    for (const out of outs) {
      if (out && ctx.fileSet.has(out)) {
        resolvedSet.add(out);
      }
    }
  }

  return [...resolvedSet].sort((a, b) => a.localeCompare(b));
}
```

### 3.2 compute-batches.mjs 变更

#### 3.2.1 extractExports() 修改

**文件路径**：`skills/understand/compute-batches.mjs`

**变更范围**：仅修改 `extractExports()` 函数（第 42-89 行），其余代码不变。

**修改策略**：优先读取 `exports-map.json`，回退到原有 tree-sitter 初始化逻辑。

```javascript
async function extractExports(projectRoot, codeFiles) {
  // 优先读取 analyze-project.mjs 产出的 exports-map.json
  const exportsMapPath = join(
    projectRoot, '.understand-anything-trae', 'intermediate', 'exports-map.json',
  );
  if (existsSync(exportsMapPath)) {
    try {
      const data = JSON.parse(readFileSync(exportsMapPath, 'utf-8'));
      if (data.exportsMap && typeof data.exportsMap === 'object') {
        // 仅保留 codeFiles 中存在的文件条目
        const codeFileSet = new Set(codeFiles.map(f => f.path));
        const filtered = new Map();
        for (const [path, symbols] of Object.entries(data.exportsMap)) {
          if (codeFileSet.has(path)) {
            filtered.set(path, symbols);
          }
        }
        process.stderr.write(
          `Info: compute-batches: loaded exports from exports-map.json ` +
          `(${filtered.size} files)\n`,
        );
        return filtered;
      }
    } catch (err) {
      process.stderr.write(
        `Warning: compute-batches: exports-map.json read failed (${err.message}) ` +
        `— falling back to tree-sitter initialization\n`,
      );
    }
  }

  // 回退：原有 tree-sitter 初始化逻辑（增量更新场景）
  process.stderr.write(
    `Warning: compute-batches: exports-map.json not found ` +
    `— falling back to tree-sitter initialization\n`,
  );

  let registry;
  try {
    const tsConfigs = builtinLanguageConfigs.filter(c => c.treeSitter);
    const tsPlugin = new TreeSitterPlugin(tsConfigs);
    await tsPlugin.init();
    registry = new PluginRegistry();
    registry.register(tsPlugin);
    const cssPlugin = new CssPlugin();
    registerAllParsers(registry, tsPlugin, cssPlugin);
  } catch (err) {
    process.stderr.write(
      `Warning: compute-batches: tree-sitter init failed (${err.message}) ` +
      `— all symbols=[] in neighborMap — cross-batch edges limited to file-level\n`,
    );
    return new Map(codeFiles.map(f => [f.path, []]));
  }

  const exportsByPath = new Map();
  for (const file of codeFiles) {
    const abs = join(projectRoot, file.path);
    let content;
    try {
      content = readFileSync(abs, 'utf-8');
    } catch (err) {
      process.stderr.write(
        `Warning: compute-batches: exports extraction failed for ${file.path} ` +
        `(read error: ${err.message}) — symbols=[] in neighborMap — ` +
        `cross-batch edges to this file limited to file-level\n`,
      );
      exportsByPath.set(file.path, []);
      continue;
    }
    try {
      const analysis = registry.analyzeFile(file.path, content);
      const names = (analysis?.exports || []).map(e => e.name).filter(Boolean);
      exportsByPath.set(file.path, names);
    } catch (err) {
      process.stderr.write(
        `Warning: compute-batches: exports extraction failed for ${file.path} ` +
        `(analyze error: ${err.message}) — symbols=[] in neighborMap — ` +
        `cross-batch edges to this file limited to file-level\n`,
      );
      exportsByPath.set(file.path, []);
    }
  }
  return exportsByPath;
}
```

**关键设计决策**：

1. **为什么保留回退逻辑**：增量更新场景下，可能只重新运行了 `scan-project.mjs` 而未运行 `analyze-project.mjs`，此时 `exports-map.json` 不存在。回退逻辑确保 `compute-batches.mjs` 可独立运行。

2. **为什么过滤 codeFiles**：`exports-map.json` 包含所有文件的导出信息，但 `compute-batches.mjs` 仅需要 `codeFiles` 的导出。过滤确保一致性。

3. **为什么使用 `Warning:` 前缀**：回退到 tree-sitter 初始化意味着性能退化，应通过 Phase 7 的 `$PHASE_WARNINGS` 机制通知用户。

4. **为什么 `exports-map.json` 读取失败也回退**：文件可能损坏或格式异常，回退到 tree-sitter 确保功能不中断。

新增格式：

